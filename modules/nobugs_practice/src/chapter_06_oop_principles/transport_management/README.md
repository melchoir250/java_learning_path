# Задача: Управление транспортом (Абстракция + Переопределение)

## Описание
Создание иерархии транспортных средств с общим поведением (движение) и индивидуальной логикой для каждого вида транспорта. Задача демонстрирует **абстракцию** (выделение общего интерфейса в базовом классе) и **переопределение** (каждый наследник реализует движение по-своему). Также важно продумать, как хранить и обновлять общее для всех состояние — пройденное расстояние.

## Условие
1.  Создайте **базовый класс** `Transport`:
    *   Поле `protected double distanceTraveled` (пройденное расстояние, км).
    *   Конструктор для инициализации начального расстояния (обычно 0).
    *   Метод `double getSpeed()` — возвращает скорость транспорта (км/ч). Может быть абстрактным или возвращать 0.
    *   Метод `void move(int hours)` — симулирует движение в течение указанного количества часов. Увеличивает `distanceTraveled` на `speed * hours`.
    *   Метод `double getDistanceTraveled()` — возвращает текущее пройденное расстояние.
2.  Создайте трех наследников с разными характеристиками и логикой:
    *   **`Bicycle`** (велосипед):
        *   Скорость: 15-25 км/ч (можно фиксированная, например 20 км/ч).
        *   При движении выводить сообщение: `"Велосипед едет по велодорожке со скоростью X км/ч"`.
    *   **`Car`** (автомобиль):
        *   Скорость: 60-120 км/ч (можно фиксированная, например 90 км/ч).
        *   При движении выводить сообщение: `"Автомобиль едет по шоссе со скоростью X км/ч"`.
    *   **`Train`** (поезд):
        *   Скорость: 100-200 км/ч (можно фиксированная, например 150 км/ч).
        *   При движении выводить сообщение: `"Поезд следует по рельсам со скоростью X км/ч"`.
3.  В методе `main` продемонстрируйте:
    *   Создание разных транспортных средств.
    *   Перемещение каждого на разное время.
    *   Вывод итогового пройденного расстояния для каждого.
    *   (Опционально) Соревнование: кто проедет больше за одинаковое время.

## Пояснение
Абстракция позволяет нам работать с любым транспортом через общий интерфейс класса `Transport`. Мы можем положить велосипед, машину и поезд в один массив и вызвать у каждого `move(5)`, не задумываясь о том, как именно они двигаются. Каждый наследник **переопределяет** поведение (уникальное сообщение) и использует общую логику увеличения пробега. Хранение `distanceTraveled` в базовом классе позволяет избежать дублирования кода.

## Пример кода (структура)

```java
// Базовый класс Transport
abstract class Transport {
    protected double distanceTraveled; // км
    protected String name;
    
    public Transport(String name) {
        this.name = name;
        this.distanceTraveled = 0;
    }
    
    public abstract double getSpeed(); // км/ч
    
    public void move(int hours) {
        double speed = getSpeed();
        double distance = speed * hours;
        distanceTraveled += distance;
        System.out.printf("%s движется %d ч. со скоростью %.1f км/ч. Пройдено: %.1f км\n",
                name, hours, speed, distance);
    }
    
    public double getDistanceTraveled() {
        return distanceTraveled;
    }
    
    public String getName() {
        return name;
    }
}

// Велосипед
class Bicycle extends Transport {
    private static final double SPEED = 20.0; // км/ч
    
    public Bicycle() {
        super("Велосипед");
    }
    
    @Override
    public double getSpeed() {
        return SPEED;
    }
}

// Автомобиль
class Car extends Transport {
    private static final double SPEED = 90.0; // км/ч
    
    public Car() {
        super("Автомобиль");
    }
    
    @Override
    public double getSpeed() {
        return SPEED;
    }
}

// Поезд
class Train extends Transport {
    private static final double SPEED = 150.0; // км/ч
    
    public Train() {
        super("Поезд");
    }
    
    @Override
    public double getSpeed() {
        return SPEED;
    }
}

// Демонстрация
public class Main {
    public static void main(String[] args) {
        // Создаём разные транспортные средства
        Transport bicycle = new Bicycle();
        Transport car = new Car();
        Transport train = new Train();
        
        System.out.println("=== Движение транспорта ===\n");
        
        // Каждый движется своё время
        bicycle.move(3);  // 3 часа
        car.move(2);      // 2 часа
        train.move(1);    // 1 час
        
        System.out.println("\n=== Итоговый пробег ===");
        System.out.printf("%s проехал: %.1f км\n", bicycle.getName(), bicycle.getDistanceTraveled());
        System.out.printf("%s проехал: %.1f км\n", car.getName(), car.getDistanceTraveled());
        System.out.printf("%s проехал: %.1f км\n", train.getName(), train.getDistanceTraveled());
        
        System.out.println("\n=== Соревнование: 4 часа в пути ===");
        
        // Массив транспорта для полиморфной обработки
        Transport[] transports = {new Bicycle(), new Car(), new Train()};
        
        for (Transport t : transports) {
            t.move(4);
        }
        
        System.out.println("\n=== Результаты соревнования ===");
        Transport winner = transports[0];
        for (Transport t : transports) {
            System.out.printf("%s: %.1f км\n", t.getName(), t.getDistanceTraveled());
            if (t.getDistanceTraveled() > winner.getDistanceTraveled()) {
                winner = t;
            }
        }
        System.out.printf("\nПобедитель: %s с результатом %.1f км!\n", 
                winner.getName(), winner.getDistanceTraveled());
    }
}
```

## Ключевые моменты реализации

*   **Абстрактный базовый класс**: `Transport` определяет общее состояние (`distanceTraveled`) и общее поведение (`move()`), но оставляет `getSpeed()` абстрактным — каждый наследник обязан задать свою скорость.
*   **Переиспользование кода**: Метод `move()` реализован один раз в базовом классе и работает для всех наследников благодаря полиморфному вызову `getSpeed()`.
*   **Инкапсуляция состояния**: Поле `distanceTraveled` защищено (`protected`), чтобы наследники могли к нему обращаться, но внешний код получает доступ только через геттер.
*   **Полиморфизм в действии**: В массиве `Transport[]` мы можем хранить любые объекты-наследники и единообразно вызывать `move()` и `getDistanceTraveled()`.
