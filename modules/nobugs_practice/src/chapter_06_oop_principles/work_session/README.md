# Задача 38: Учёт рабочего времени (Инкапсуляция + Расчёт)

## Описание проекта

Разработайте систему для учёта рабочего времени, демонстрирующую принципы инкапсуляции и инкапсуляции логики над
внутренним состоянием. Класс `WorkSession` хранит приватные поля времени начала и окончания сессии, предоставляя методы
для управления сессией и расчёта продолжительности. Вся логика работы с временем скрыта внутри класса.

## Структура проекта

```
work_session/
├── WorkSession.java          # Класс рабочей сессии
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `WorkSession`

Представляет рабочую сессию с учётом времени:

**Поля (все приватные)**:

- `LocalDateTime startTime` - время начала сессии
- `LocalDateTime endTime` - время окончания сессии
- `String description` - описание сессии (опционально)
- `String sessionId` - уникальный идентификатор сессии (опционально)

**Конструкторы**:

- `WorkSession()` - создаёт сессию с текущим временем начала
- `WorkSession(String description)` - создаёт сессию с описанием и текущим временем
- `WorkSession(LocalDateTime startTime)` - создаёт сессию с указанным временем начала

**Методы**:

#### `void start()`

- Начинает сессию, устанавливая `startTime` на текущий момент
- Если сессия уже начата, может выбрасывать исключение или игнорировать вызов

#### `void end()`

- Завершает сессию, устанавливая `endTime` на текущий момент
- Если сессия не начата, выбрасывает исключение
- Если сессия уже завершена, выбрасывает исключение или игнорирует

#### `long getDuration()`

- Возвращает продолжительность сессии в минутах (или секундах, часах)
- Если сессия не завершена, рассчитывает продолжительность до текущего момента
- Если сессия не начата, возвращает 0

#### `long getDuration(TimeUnit unit)`

- Возвращает продолжительность в указанной единице измерения (часы, минуты, секунды)

#### `boolean isActive()`

- Возвращает `true`, если сессия начата, но ещё не завершена

#### `boolean isCompleted()`

- Возвращает `true`, если сессия завершена

#### `LocalDateTime getStartTime()`

- Возвращает время начала (геттер)

#### `LocalDateTime getEndTime()`

- Возвращает время окончания (геттер, может быть null)

#### `String getDescription()`

- Возвращает описание сессии

#### `void setDescription(String description)`

- Устанавливает описание сессии (опционально)

### 2. Класс `Main`

Демонстрирует работу системы:

- Создаёт рабочие сессии
- Показывает начало и завершение сессий
- Вычисляет продолжительность
- Демонстрирует работу с активными и завершёнными сессиями

## Технические требования

### Инкапсуляция

- Поля времени должны быть приватными
- Нет прямого доступа к полям извне
- Вся логика расчёта времени внутри класса

### Работа с временем

- Использование `java.time.LocalDateTime`
- Корректный расчёт разницы между временами
- Учёт различных единиц измерения

### Валидация

- Нельзя завершить незапущенную сессию
- Нельзя запустить уже запущенную сессию
- Нельзя перезаписать время начала или окончания

## Ключевые концепции

- **Инкапсуляция**: скрытие внутреннего состояния
- **Логика над состоянием**: методы расчёта на основе приватных полей
- **Управление жизненным циклом**: начало и завершение сессии
- **Неизменяемость после завершения**: время окончания не меняется
