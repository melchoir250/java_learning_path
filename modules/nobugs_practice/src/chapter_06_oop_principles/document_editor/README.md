# Задача 25: Редактируемый документ (Инкапсуляция + История изменений)

## Описание проекта

Разработайте систему редактируемого документа, демонстрирующую принципы инкапсуляции и управления историей изменений.
Документ хранит своё содержимое в приватном поле, предоставляет методы для изменения текста и отмены последних операций.
История изменений управляется внутри класса, что позволяет отслеживать состояние документа и возвращаться к предыдущим
версиям.

## Структура проекта

```
document_editor/
├── Document.java             # Класс документа с историей изменений
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `Document`

Представляет редактируемый документ с поддержкой отмены действий:

**Поля (все приватные)**:

- `StringBuilder content` - текущее содержимое документа (или `String`)
- `Stack<String> history` - стек для хранения истории изменений
- `int maxHistorySize` - максимальный размер истории (опционально)

**Конструктор**:

- Создаёт пустой документ
- Инициализирует историю изменений
- Сохраняет начальное состояние (пустую строку) в историю

**Методы**:

#### `void appendText(String text)`

- Добавляет указанный текст в конец документа
- Сохраняет состояние документа **до** изменения в историю
- Обновляет содержимое документа
- Проверяет, что текст не пустой (опционально)

#### `void insertText(int position, String text)`

- Вставляет текст в указанную позицию (опционально)
- Сохраняет состояние до изменения

#### `void deleteText(int length)`

- Удаляет указанное количество символов с конца (опционально)
- Сохраняет состояние до изменения

#### `void undo()`

- Отменяет последнее изменение документа
- Восстанавливает предыдущее состояние из истории
- Если история пуста, ничего не делает или выбрасывает исключение
- Возвращает булево значение успешности отмены (опционально)

#### `boolean canUndo()`

- Возвращает `true`, если есть изменения, которые можно отменить

#### `String getContent()`

- Возвращает текущее содержимое документа
- Возвращает копию или ссылку на содержимое

#### `void clearHistory()`

- Очищает историю изменений (опционально)
- Сохраняет текущее состояние как базовое

#### `int getHistorySize()`

- Возвращает количество сохранённых состояний в истории

### 2. Класс `Main`

Демонстрирует работу системы:

- Создаёт документ
- Выполняет несколько операций добавления текста
- Показывает содержимое после каждой операции
- Выполняет отмену последних изменений
- Демонстрирует, что нельзя отменить больше, чем было изменений
- Показывает корректное восстановление состояний

## Технические требования

### Инкапсуляция

- Содержимое документа доступно только через методы
- История изменений полностью скрыта внутри класса
- Внутреннее состояние защищено от прямого доступа

### Управление состоянием

- Использование стека (`Stack` или `Deque`) для хранения истории
- Каждое изменение сохраняет состояние **до** применения
- Отмена восстанавливает предыдущее состояние

### Обработка граничных случаев

- Попытка отмены при пустой истории не вызывает ошибку
- Добавление пустого текста может игнорироваться
- Вставка в неверную позицию обрабатывается корректно

## Ключевые концепции

- **Инкапсуляция**: сокрытие внутреннего состояния документа
- **Управление историей**: паттерн "Команда" или "Memento" в упрощённом виде
- **Стек**: структура данных для LIFO (последним пришёл — первым ушёл)
- **Неизменяемость**: сохранение состояний для возможности отката
