# Задача 33: Книга с метаданными (Инкапсуляция)

## Описание проекта

Разработайте класс книги, демонстрирующий принцип инкапсуляции и неизменяемости (immutability). Все поля книги доступны
только для чтения через геттеры и не могут быть изменены после создания объекта. Это обеспечивает защиту данных и
предсказуемость поведения.

## Структура проекта

```
book_metadata/
├── Book.java                 # Класс книги с неизменяемыми полями
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `Book`

Представляет книгу с защищёнными метаданными:

- **Поля** (все приватные):
    - `String title` - название книги
    - `String author` - автор книги
    - `String isbn` - международный стандартный книжный номер
    - `int year` - год издания (опционально)
    - `String publisher` - издательство (опционально)
    - `int pageCount` - количество страниц (опционально)
- **Конструктор**: принимает все поля и инициализирует объект
- **Геттеры**: для доступа к каждому полю
    - `String getTitle()` - возвращает название
    - `String getAuthor()` - возвращает автора
    - `String getIsbn()` - возвращает ISBN
    - `int getYear()` - возвращает год издания (опционально)
    - `String getPublisher()` - возвращает издательство (опционально)
    - `int getPageCount()` - возвращает количество страниц (опционально)
- **Нет публичных сеттеров** - поля нельзя изменить после создания

### 2. Класс `Main`

Демонстрирует работу системы:

- Создаёт объекты книг через конструктор
- Показывает, что поля доступны только через геттеры
- Демонстрирует, что изменить поля после создания невозможно
- Показывает преимущества неизменяемости

## Технические требования

### Инкапсуляция

- Все поля объявлены как `private`
- Нет публичных сеттеров
- Доступ к полям только через геттеры

### Неизменяемость (Immutability)

- Объект создаётся один раз и не может быть изменён
- Все поля являются final (опционально, но рекомендуется)
- Класс должен быть объявлен как `final` (опционально)

### Валидация (опционально)

- В конструкторе проверять корректность ISBN (формат, длина)
- Проверять, что название и автор не пустые
- Проверять, что год издания в разумных пределах

## Ключевые концепции

- **Инкапсуляция**: скрытие внутреннего состояния объекта
- **Неизменяемость (Immutability)**: объект не может быть изменён после создания
- **Защита данных**: предотвращение несанкционированных изменений
- **Безопасность**: предсказуемое поведение объекта
