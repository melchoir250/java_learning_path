# Задача 18: Информация о книгах (Инкапсуляция + Связанность)

## Описание проекта

Разработайте систему управления библиотекой, демонстрирующую принципы инкапсуляции и слабой связанности. Класс книги
должен скрывать свои данные, предоставляя доступ только через геттеры, а библиотека должна предоставлять удобный
интерфейс для работы с книгами, не раскрывая внутреннюю структуру хранения.

## Структура проекта

```
library_info/
├── Book.java           # Класс книги с инкапсулированными полями
├── Library.java        # Класс библиотеки для управления книгами
├── Main.java           # Демонстрация работы
└── README.md           # Описание задачи
```

## Функциональные требования

### 1. Класс `Book`

Представляет книгу с инкапсулированными данными:

- **Поля** (все должны быть `private`):
    - `String title` - название книги
    - `String author` - автор книги
    - `int year` - год издания
- **Конструктор**: для инициализации всех полей
- **Геттеры**: для доступа к каждому полю (`getTitle()`, `getAuthor()`, `getYear()`)
- **Сеттеры**: опционально, но для инкапсуляции лучше не предоставлять или предоставить только при необходимости
- **Метод `toString()`**: для удобного вывода информации о книге

**Важно**: Поля должны быть доступны только через геттеры. Никакого прямого доступа к полям извне класса.

### 2. Класс `Library`

Представляет библиотеку с коллекцией книг:

- **Поле**: коллекция для хранения книг (например, `List<Book>`)
- **Конструктор**: инициализирует пустую коллекцию
- **Методы**:
    - `void addBook(Book book)` - добавляет книгу в библиотеку
    - `List<Book> findBooksByAuthor(String author)` - возвращает все книги указанного автора
    - `List<Book> findBooksByYear(int year)` - возвращает все книги, изданные в указанном году
    - `List<Book> findBooksByYearRange(int fromYear, int toYear)` - возвращает книги, изданные в диапазоне лет (
      опционально)
    - `List<Book> getAllBooks()` - возвращает копию списка всех книг (не оригинальную коллекцию)
    - `int getBookCount()` - возвращает количество книг в библиотеке
    - `boolean removeBook(String title)` - удаляет книгу по названию (опционально)

### 3. Класс `Main`

Демонстрирует работу системы:

- Создаёт несколько книг
- Добавляет их в библиотеку
- Выполняет поиск по автору
- Выполняет поиск по году
- Выводит результаты поиска
- Показывает, что доступ к данным книг осуществляется только через геттеры

## Технические требования

### Инкапсуляция

- Все поля класса `Book` должны быть `private`
- Доступ к данным только через геттеры
- Внутренняя структура хранения библиотеки скрыта от внешнего кода

### Слабая связанность

- Класс `Library` не зависит от внутренней реализации `Book`
- Внешний код не имеет прямого доступа к коллекции книг
- Возврат копий коллекций вместо оригиналов для защиты данных

### Поиск и фильтрация

- Использование Stream API для поиска книг (рекомендуется)
- Или использование циклов для демонстрации базового подхода

## Ключевые концепции

- **Инкапсуляция**: скрытие внутреннего состояния объекта
- **Слабая связанность**: взаимодействие через методы, а не через прямое обращение к данным
- **Защита данных**: возврат копий коллекций вместо оригиналов
- **Чистый интерфейс**: предоставление методов для всех необходимых операций

## Дополнительные требования (опционально)

1. **Защита от дубликатов**:
    - При добавлении книги проверять, нет ли уже такой книги в библиотеке
    - Использовать `equals()` и `hashCode()` в классе `Book`

2. **Сортировка**:
    - Добавить метод для получения книг, отсортированных по году издания
    - Или по названию

3. **Статистика**:
    - Метод для получения самого популярного автора
    - Метод для получения количества книг по каждому автору

4. **Поиск по части названия**:
    - `findBooksByTitleKeyword(String keyword)` - поиск книг, содержащих ключевое слово в названии

5. **Сохранение и загрузка**:
    - Сохранение коллекции книг в файл
    - Загрузка книг из файла

6. **Жанры**:
    - Добавить поле `genre` в класс `Book`
    - Реализовать поиск по жанру

## Критерии оценки реализации

1. **Инкапсуляция**:
    - Все поля класса `Book` приватные ✓
    - Доступ только через геттеры ✓
    - Нет прямого доступа к полям из других классов

2. **Защита коллекции**:
    - Метод `getAllBooks()` возвращает копию списка
    - Внешний код не может изменить внутреннюю коллекцию библиотеки

3. **Связанность**:
    - Классы взаимодействуют только через методы
    - Внутренняя структура библиотеки скрыта

4. **Функциональность**:
    - Корректная работа всех методов поиска
    - Обработка пограничных случаев (автор не найден, год без книг)

## Как должен работать код

1. Создаются объекты `Book` с различными названиями, авторами и годами
2. Книги добавляются в `Library` через метод `addBook()`
3. Поиск по автору возвращает список книг этого автора
4. Поиск по году возвращает список книг, изданных в этот год
5. Все операции выполняются через методы, без прямого доступа к полям
6. Информация о книгах выводится с использованием геттеров
