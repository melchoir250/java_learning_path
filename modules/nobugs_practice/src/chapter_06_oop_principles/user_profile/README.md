# Задача 40: Профили и настройки (Композиция)

## Описание проекта

Разработайте систему для управления пользовательскими профилями, демонстрирующую принцип композиции. Класс `UserProfile`
содержит компоненты `Settings` и `Preferences`, делегируя им соответствующие методы доступа. Вместо того чтобы хранить
все поля напрямую, профиль собирается из независимых компонентов, что упрощает поддержку и расширение системы.

## Структура проекта

```
user_profile/
├── UserProfile.java          # Класс профиля пользователя (композиция)
├── Settings.java             # Класс настроек (тема, язык, уведомления)
├── Preferences.java          # Класс предпочтений (категории, избранное)
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `Settings` (Настройки)

Представляет системные настройки пользователя:

- **Поля**:
    - `String theme` - тема оформления ("light", "dark", "auto")
    - `String language` - язык интерфейса ("ru", "en", "de")
    - `boolean notificationsEnabled` - включены ли уведомления
    - `boolean soundEnabled` - включены ли звуки (опционально)
    - `int fontSize` - размер шрифта (опционально)
- **Конструктор**: инициализирует настройки значениями по умолчанию
- **Геттеры и сеттеры**: для доступа к каждому полю
- **Метод `resetToDefaults()`**: сброс к настройкам по умолчанию
- **Метод `toJson()`**: экспорт в JSON (опционально)

### 2. Класс `Preferences` (Предпочтения)

Представляет пользовательские предпочтения:

- **Поля**:
    - `List<String> favoriteCategories` - избранные категории
    - `List<String> favoriteItems` - избранные элементы (опционально)
    - `Map<String, String> customSettings` - пользовательские настройки (опционально)
- **Конструктор**: инициализирует пустые списки
- **Методы**:
    - `void addFavoriteCategory(String category)` - добавить категорию
    - `void removeFavoriteCategory(String category)` - удалить категорию
    - `boolean isFavoriteCategory(String category)` - проверка
    - `List<String> getFavoriteCategories()` - получить копию списка
    - `void clearFavorites()` - очистить избранное

### 3. Класс `UserProfile`

Представляет профиль пользователя (композиция):

- **Поля**:
    - `String username` - имя пользователя
    - `String email` - email (опционально)
    - `Settings settings` - настройки (композиция)
    - `Preferences preferences` - предпочтения (композиция)
- **Конструктор**: принимает имя пользователя и создаёт компоненты
- **Методы доступа к настройкам** (делегирование):
    - `String getTheme()` / `void setTheme(String theme)`
    - `String getLanguage()` / `void setLanguage(String language)`
    - `boolean isNotificationsEnabled()` / `void setNotificationsEnabled(boolean enabled)`
- **Методы доступа к предпочтениям** (делегирование):
    - `void addFavoriteCategory(String category)`
    - `void removeFavoriteCategory(String category)`
    - `boolean isFavoriteCategory(String category)`
    - `List<String> getFavoriteCategories()`
- **Методы управления профилем**:
    - `void resetSettingsToDefaults()` - сброс настроек
    - `String getProfileInfo()` - получить информацию о профиле

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт профиль пользователя
- Изменяет настройки и предпочтения через методы профиля
- Показывает, что профиль делегирует вызовы компонентам
- Демонстрирует преимущества композиции

## Технические требования

### Композиция

- `UserProfile` содержит экземпляры `Settings` и `Preferences`
- Компоненты создаются внутри профиля или передаются извне
- Профиль управляет жизненным циклом компонентов

### Делегирование

- Методы профиля делегируют вызовы соответствующим компонентам
- Внешний код работает с профилем, а не напрямую с компонентами

### Инкапсуляция

- Компоненты скрыты внутри профиля
- Доступ к настройкам только через методы профиля

### Расширяемость

- Новые компоненты легко добавляются в профиль
- Существующий код не требует изменений

## Ключевые концепции

- **Композиция**: объект состоит из других объектов
- **Делегирование**: передача ответственности компонентам
- **Разделение ответственности**: каждый компонент отвечает за свою область
- **Инкапсуляция**: скрытие внутренней структуры
