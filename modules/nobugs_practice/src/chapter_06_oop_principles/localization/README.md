# Задача 24: Мультиязычный интерфейс (Интерфейс + Стратегия)

## Описание проекта

Разработайте систему локализации, демонстрирующую паттерн "Стратегия" через интерфейсы. Система должна поддерживать
несколько языков и позволять легко переключаться между ними. Логика перевода изолирована от основной программы, что
позволяет добавлять новые языки без изменения существующего кода.

## Структура проекта

```
localization/
├── Localizable.java          # Интерфейс локализации
├── Russian.java              # Реализация для русского языка
├── English.java              # Реализация для английского языка
├── German.java               # Реализация для немецкого языка
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Localizable`

Определяет контракт для всех языковых реализаций:

- **Метод `String translate(String key)`** - возвращает перевод по ключу
- **Метод `String getLanguageName()`** - возвращает название языка (опционально)
- **Метод `String getLanguageCode()`** - возвращает код языка (например, "ru", "en", "de") (опционально)

### 2. Реализации локализации

#### `Russian` (Русский язык)

- Содержит словарь переводов на русский язык
- **Структура**: `Map<String, String>` или `Properties` для хранения переводов
- **Ключи и значения** (примеры):
    - `"greeting"` → `"Здравствуйте"`
    - `"farewell"` → `"До свидания"`
    - `"yes"` → `"Да"`
    - `"no"` → `"Нет"`
    - `"save"` → `"Сохранить"`
    - `"cancel"` → `"Отмена"`
    - `"error"` → `"Ошибка"`
    - `"success"` → `"Успешно"`

#### `English` (Английский язык)

- Содержит словарь переводов на английский язык
- **Ключи и значения** (примеры):
    - `"greeting"` → `"Hello"`
    - `"farewell"` → `"Goodbye"`
    - `"yes"` → `"Yes"`
    - `"no"` → `"No"`
    - `"save"` → `"Save"`
    - `"cancel"` → `"Cancel"`
    - `"error"` → `"Error"`
    - `"success"` → `"Success"`

#### `German` (Немецкий язык)

- Содержит словарь переводов на немецкий язык
- **Ключи и значения** (примеры):
    - `"greeting"` → `"Hallo"`
    - `"farewell"` → `"Auf Wiedersehen"`
    - `"yes"` → `"Ja"`
    - `"no"` → `"Nein"`
    - `"save"` → `"Speichern"`
    - `"cancel"` → `"Abbrechen"`
    - `"error"` → `"Fehler"`
    - `"success"` → `"Erfolg"`

### 3. Класс `Main`

Демонстрирует работу системы:

- Создаёт экземпляры всех языковых реализаций
- Показывает переводы одних и тех же ключей на разных языках
- Демонстрирует переключение между языками
- Обрабатывает отсутствие перевода для ключа

## Технические требования

### Паттерн "Стратегия"

- Языковая стратегия может быть выбрана во время выполнения
- Логика перевода изолирована от основной программы
- Легкое переключение между языками

### Полиморфизм через интерфейс

- Все языки реализуют единый интерфейс `Localizable`
- Основная программа работает с интерфейсом, а не с конкретными классами

### Инкапсуляция

- Каждый класс языка содержит свой словарь переводов
- Детали хранения переводов скрыты внутри класса

### Обработка отсутствия перевода

- Если ключ не найден, метод `translate()` должен возвращать:
    - Сам ключ (например, `"greeting"`)
    - Или сообщение об ошибке (например, `"[не найдено: greeting]"`)
    - Или значение по умолчанию
