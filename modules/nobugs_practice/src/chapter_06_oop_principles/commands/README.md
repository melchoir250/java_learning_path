# Задача 22: Кнопка с действием (Command Pattern + Полиморфизм)

## Описание проекта

Разработайте систему, демонстрирующую паттерн "Команда" (Command), где действия инкапсулированы в объекты команд. Кнопка
может выполнять любую команду, не зная о её конкретной реализации. Это иллюстрирует принцип "поведение как объект",
позволяя отделить инициатора действия от самого действия.

## Структура проекта

```
command_pattern/
├── Command.java              # Интерфейс команды
├── PrintCommand.java         # Команда печати сообщения
├── SaveCommand.java          # Команда сохранения данных
├── ExitCommand.java          # Команда выхода из программы
├── Button.java               # Кнопка (инициатор команды)
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Command`

Определяет контракт для всех команд:

- **Метод `void execute()`** - выполняет команду
- **Метод `void undo()`** - отменяет команду (опционально)
- **Метод `String getDescription()`** - возвращает описание команды (опционально)

### 2. Реализации команд

#### `PrintCommand`

- Выводит сообщение на экран
- **Поля**:
    - `String message` - сообщение для вывода
    - `int repeatCount` - количество повторений (опционально)
- **Логика**:
    - При выполнении выводит сообщение в консоль
    - Может поддерживать форматирование (время, префикс)

#### `SaveCommand`

- Сохраняет данные в файл или базу данных
- **Поля**:
    - `String data` - данные для сохранения
    - `String filename` - имя файла (опционально)
- **Логика**:
    - При выполнении сохраняет данные
    - Может проверять существование файла
    - Обрабатывает ошибки записи

#### `ExitCommand`

- Завершает работу программы
- **Поля**:
    - `int exitCode` - код возврата (0 - успешное завершение)
- **Логика**:
    - При выполнении выводит прощальное сообщение
    - Завершает приложение с указанным кодом
    - Может выполнять очистку ресурсов

### 3. Класс `Button`

Представляет кнопку, которая может выполнять команду:

- **Поле**:
    - `Command command` - команда, назначенная на кнопку
- **Конструктор**: может принимать команду или создаваться без команды
- **Методы**:
    - `void setCommand(Command command)` - назначить команду на кнопку
    - `void press()` - выполнить назначенную команду
    - `void pressWithHistory()` - выполнить с добавлением в историю (опционально)

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт различные команды
- Создаёт кнопку
- Назначает разные команды на одну и ту же кнопку
- Показывает, что кнопка не знает о внутренней логике команд
- Демонстрирует полиморфное поведение

## Технические требования

### Паттерн "Команда" (Command)

- Каждое действие инкапсулировано в отдельный объект команды
- Инициатор (Button) не знает о реализации команды
- Команды могут быть параметризованы данными через конструктор

### Полиморфизм

- Все команды реализуют единый интерфейс `Command`
- `Button` работает с типом `Command`, а не с конкретными командами

### Инкапсуляция поведения

- Поведение вынесено из кнопки в отдельные объекты
- Кнопка отвечает только за вызов команды

### История команд (опционально)

- Добавление возможности отмены (`undo`)
- Хранение истории выполненных команд

## Ключевые концепции

- **Command Pattern**: инкапсуляция запроса как объекта
- **Полиморфизм**: единый интерфейс для разных команд
- **Разделение ответственности**: инициатор и исполнитель разделены
- **Поведение как объект**: действия представлены объектами

## Дополнительные требования (опционально)

1. **История команд**:
    - Класс `CommandHistory` для хранения выполненных команд
    - Метод `undo()` для отмены последней команды
    - Метод `redo()` для повторного выполнения

2. **Макрокоманды**:
    - Класс `MacroCommand`, реализующий `Command`
    - Содержит список команд и выполняет их последовательно
    - Позволяет группировать действия

3. **Параметризованные команды**:
    - Команды, которые принимают параметры во время выполнения
    - Интерфейс с методом `execute(Object... params)`

4. **Асинхронное выполнение**:
    - Команды, выполняющиеся в отдельном потоке
    - Обработка результатов выполнения

5. **Запланированные команды**:
    - Выполнение команды с задержкой
    - Планирование на определённое время

6. **Команды с обратной связью**:
    - Метод `execute()` возвращает результат выполнения
    - Обработка успеха/неудачи

## Критерии оценки реализации

1. **Паттерн "Команда"**:
    - Каждое действие реализовано как отдельный класс команды
    - Кнопка не знает о конкретных командах

2. **Полиморфизм**:
    - Один и тот же метод `press()` выполняет разные действия
    - Команды взаимозаменяемы

3. **Инкапсуляция**:
    - Поведение инкапсулировано в объектах команд
    - Кнопка имеет единственную ответственность

4. **Гибкость**:
    - Новые команды добавляются без изменения кнопки
    - Команды могут быть скомбинированы

## Как должен работать код

### Этап 1: Базовое использование

1. Создаются объекты команд (`PrintCommand`, `SaveCommand`, `ExitCommand`)
2. Создаётся кнопка
3. На кнопку назначается одна из команд через `setCommand()`
4. При нажатии кнопки (`press()`) выполняется назначенная команда

### Этап 2: Смена поведения

1. Та же кнопка получает новую команду через `setCommand()`
2. При повторном нажатии выполняется уже другая логика
3. Код кнопки не меняется

### Этап 3: Демонстрация полиморфизма

1. Все команды хранятся в списке типа `List<Command>`
2. Одна кнопка последовательно выполняет разные команды
3. Показывается, что кнопка работает с абстракцией

```
=== Создаём кнопку ===

=== Назначаем команду печати ===

=== Меняем команду на выход ===
Нажимаем кнопку...

=== Демонстрация макрокоманды ===
Создаём макрокоманду из трёх действий:

- Печать "Шаг 2"

Нажимаем кнопку...
Выполнение макрокоманды:
  1. Шаг 1
  2. Сохранение выполнено
  3. Шаг 2
Макрокоманда завершена
```

## Принципы, демонстрируемые задачей

1. **Command Pattern**: поведение инкапсулировано в объектах, что позволяет параметризовать объекты действиями
2. **Полиморфизм через интерфейс**: единый способ вызова разных действий
3. **Разделение ответственности**: кнопка отвечает за инициацию, команды — за выполнение
4. **Открытость/закрытость**: новые команды добавляются без изменения существующего кода
5. **Поведение как объект**: возможность хранить, передавать и комбинировать действия

Выполнено повторно: Печать "Мир"
