# Задача 30: Работа с медиа (Полиморфизм + Общий интерфейс)

## Описание проекта

Разработайте систему медиа-плеера, демонстрирующую принцип полиморфизма через общий интерфейс. Различные типы
медиа-контента (песни, подкасты, аудиокниги) реализуют единый интерфейс `Playable`, что позволяет плееру воспроизводить
их единообразно, не зная о конкретном типе контента.

## Структура проекта

```
media_player/
├── Playable.java             # Интерфейс воспроизводимого контента
├── Song.java                 # Класс песни
├── Podcast.java              # Класс подкаста
├── Audiobook.java            # Класс аудиокниги
├── Player.java               # Класс плеера
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Playable`

Определяет контракт для всех воспроизводимых медиа:

- **Метод `void play()`** - воспроизводит контент
- **Метод `void pause()`** - приостанавливает воспроизведение (опционально)
- **Метод `void stop()`** - останавливает воспроизведение (опционально)
- **Метод `String getTitle()`** - возвращает название контента
- **Метод `int getDuration()`** - возвращает продолжительность в секундах (опционально)
- **Метод `String getInfo()`** - возвращает информацию о контенте (опционально)

### 2. Реализации интерфейса

#### `Song` (Песня)

- **Поля**:
    - `String title` - название песни
    - `String artist` - исполнитель
    - `String album` - альбом (опционально)
    - `int duration` - продолжительность в секундах
    - `String genre` - жанр (опционально)
- **Конструктор**: принимает название, исполнителя и продолжительность
- **Метод `play()`**: выводит информацию о воспроизведении песни
- **Метод `getTitle()`**: возвращает название песни

#### `Podcast` (Подкаст)

- **Поля**:
    - `String title` - название выпуска
    - `String host` - ведущий
    - `int episodeNumber` - номер выпуска
    - `int duration` - продолжительность
    - `String topic` - тема выпуска (опционально)
- **Конструктор**: принимает название, ведущего, номер выпуска и продолжительность
- **Метод `play()`**: выводит информацию о воспроизведении подкаста
- **Метод `getTitle()`**: возвращает название подкаста

#### `Audiobook` (Аудиокнига)

- **Поля**:
    - `String title` - название книги
    - `String author` - автор
    - `String narrator` - чтец
    - `int duration` - продолжительность
    - `int chapter` - текущая глава (опционально)
- **Конструктор**: принимает название, автора, чтеца и продолжительность
- **Метод `play()`**: выводит информацию о воспроизведении аудиокниги
- **Метод `getTitle()`**: возвращает название книги

### 3. Класс `Player`

Представляет медиа-плеер:

- **Поле**:
    - `List<Playable> playlist` - список воспроизводимого контента
- **Конструктор**: инициализирует пустой плейлист
- **Методы**:
    - `void addToPlaylist(Playable item)` - добавить медиа в плейлист
    - `void removeFromPlaylist(Playable item)` - удалить из плейлиста
    - `void playAll()` - воспроизвести весь плейлист
    - `void playShuffled()` - воспроизвести в случайном порядке (опционально)
    - `void playNext()` - воспроизвести следующий трек (опционально)
    - `List<Playable> getPlaylist()` - вернуть копию плейлиста
    - `int getTotalDuration()` - общая продолжительность плейлиста (опционально)

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт различные медиа-объекты (песни, подкасты, аудиокниги)
- Создаёт плеер
- Добавляет все объекты в плейлист
- Запускает воспроизведение всего плейлиста
- Показывает, что плеер работает с любыми объектами, реализующими `Playable`

## Технические требования

### Полиморфизм через интерфейс

- Все медиа-классы реализуют интерфейс `Playable`
- `Player` работает с типом `Playable`, а не с конкретными классами
- Метод `playAll()` единообразно вызывает `play()` для каждого элемента

### Инкапсуляция

- Каждый медиа-класс инкапсулирует свои специфические поля
- Плеер не знает о внутренней структуре медиа

### Расширяемость

- Новые типы медиа (например, `RadioShow`, `Interview`) добавляются легко
- Существующий код не требует изменений

## Ключевые концепции

- **Полиморфизм**: единый интерфейс для разных типов медиа
- **Интерфейс как контракт**: общее поведение для всех воспроизводимых объектов
- **Композиция**: плеер содержит коллекцию `Playable`
- **Слабая связанность**: плеер зависит только от интерфейса
