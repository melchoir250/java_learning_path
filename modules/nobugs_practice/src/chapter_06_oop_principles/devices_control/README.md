# Задача 37: Устройства с разными функциями (Полиморфизм)

## Описание проекта

Разработайте систему управления умными устройствами, демонстрирующую принцип полиморфизма через общий интерфейс.
Различные устройства (лампа, вентилятор, обогреватель) реализуют единый интерфейс `Device`. Контроллер управляет любыми
устройствами через этот интерфейс, не зная их конкретного типа. Это позволяет легко добавлять новые устройства без
изменения существующего кода.

## Структура проекта

```
devices_control/
├── Device.java               # Интерфейс устройства
├── Lamp.java                 # Реализация лампы
├── Fan.java                  # Реализация вентилятора
├── Heater.java               # Реализация обогревателя
├── DeviceController.java     # Контроллер управления
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Device`

Определяет контракт для всех управляемых устройств:

- **Метод `void turnOn()`** - включает устройство
- **Метод `void turnOff()`** - выключает устройство
- **Метод `boolean isOn()`** - возвращает текущее состояние
- **Метод `String getName()`** - возвращает название устройства
- **Метод `String getStatus()`** - возвращает строковое описание состояния (опционально)

### 2. Реализации устройств

#### `Lamp` (Лампа)

- **Поля**:
    - `String name` - название/расположение лампы
    - `boolean isOn` - состояние включения
    - `int brightness` - яркость (0-100) (опционально)
    - `String color` - цвет свечения (опционально)
- **Конструктор**: принимает название
- **Метод `turnOn()`**: включает лампу с последней яркостью или по умолчанию
- **Метод `turnOff()`**: выключает лампу
- **Специфические методы** (опционально):
    - `void setBrightness(int level)` - установить яркость
    - `void setColor(String color)` - установить цвет

#### `Fan` (Вентилятор)

- **Поля**:
    - `String name` - название/расположение вентилятора
    - `boolean isOn` - состояние включения
    - `int speed` - скорость вращения (0-5) (опционально)
    - `boolean oscillating` - режим поворота (опционально)
- **Конструктор**: принимает название
- **Метод `turnOn()`**: включает вентилятор с последней скоростью
- **Метод `turnOff()`**: выключает вентилятор
- **Специфические методы** (опционально):
    - `void setSpeed(int speed)` - установить скорость
    - `void toggleOscillation()` - включить/выключить поворот

#### `Heater` (Обогреватель)

- **Поля**:
    - `String name` - название/расположение обогревателя
    - `boolean isOn` - состояние включения
    - `int temperature` - температура нагрева (опционально)
    - `int mode` - режим работы (эконом, нормальный, мощный) (опционально)
- **Конструктор**: принимает название
- **Метод `turnOn()`**: включает обогреватель с последней температурой
- **Метод `turnOff()`**: выключает обогреватель
- **Специфические методы** (опционально):
    - `void setTemperature(int temp)` - установить температуру
    - `void setMode(int mode)` - установить режим

### 3. Класс `DeviceController`

Представляет контроллер для управления устройствами:

- **Поле**:
    - `List<Device> devices` - список управляемых устройств
- **Конструктор**: инициализирует пустой список
- **Методы**:
    - `void addDevice(Device device)` - добавить устройство
    - `void removeDevice(Device device)` - удалить устройство
    - `void turnAllOn()` - включить все устройства
    - `void turnAllOff()` - выключить все устройства
    - `void turnOnDevice(String name)` - включить устройство по имени
    - `void turnOffDevice(String name)` - выключить устройство по имени
    - `void showStatus()` - показать статус всех устройств
    - `List<Device> getDevices()` - вернуть копию списка устройств (опционально)

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт различные устройства (лампа, вентилятор, обогреватель)
- Создаёт контроллер
- Добавляет устройства в контроллер
- Управляет устройствами через контроллер
- Показывает, что контроллер не знает о конкретных типах устройств
- Демонстрирует полиморфное поведение

## Технические требования

### Полиморфизм через интерфейс

- Все устройства реализуют интерфейс `Device`
- `DeviceController` работает с типом `Device`, а не с конкретными классами
- Один и тот же код управляет разными типами устройств

### Инкапсуляция

- Каждое устройство инкапсулирует свои специфические поля
- Состояние устройства защищено и изменяется только через методы

### Расширяемость

- Новые устройства (например, `AirConditioner`, `Curtains`) добавляются легко
- Существующий код не требует изменений

### Управление состоянием

- Корректное отслеживание состояния каждого устройства
- Метод `isOn()` возвращает актуальное состояние

## Ключевые концепции

- **Полиморфизм**: единый интерфейс для разных типов устройств
- **Интерфейс как контракт**: общее поведение для всех устройств
- **Слабая связанность**: контроллер зависит только от интерфейса
- **Открытость/закрытость**: новые устройства добавляются без изменения контроллера
