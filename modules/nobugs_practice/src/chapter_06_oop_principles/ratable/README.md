# Задача 27: Рейтинг и фильтрация (Полиморфизм)

## Описание проекта

Разработайте систему для работы с рейтингами различных типов объектов, демонстрирующую полиморфизм через интерфейс.
Книги, фильмы и курсы имеют общее поведение — возможность получить рейтинг. Сервис фильтрации может работать с любым
объектом, реализующим этот интерфейс, позволяя отбирать лучшие элементы независимо от их типа.

## Структура проекта

```
ratable/
├── Ratable.java              # Интерфейс для объектов с рейтингом
├── Book.java                 # Класс книги
├── Movie.java                # Класс фильма
├── Course.java               # Класс курса
├── RatingService.java        # Сервис для фильтрации и работы с рейтингами
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Ratable`

Определяет контракт для всех объектов, имеющих рейтинг:

- **Метод `double getRating()`** - возвращает числовой рейтинг объекта
- **Метод `String getName()`** - возвращает название объекта (опционально)
- **Метод `String getDescription()`** - возвращает описание (опционально)

### 2. Реализации интерфейса

#### `Book` (Книга)

- **Поля**:
    - `String title` - название книги
    - `String author` - автор
    - `int year` - год издания
    - `double rating` - рейтинг (от 0 до 5 или от 0 до 10)
    - `int reviewCount` - количество отзывов (опционально)
- **Конструктор**: принимает все или основные поля
- **Метод `getRating()`**: возвращает рейтинг книги
- **Метод `getName()`**: возвращает название книги

#### `Movie` (Фильм)

- **Поля**:
    - `String title` - название фильма
    - `String director` - режиссёр
    - `int year` - год выпуска
    - `double rating` - рейтинг
    - `String genre` - жанр (опционально)
- **Конструктор**: принимает все или основные поля
- **Метод `getRating()`**: возвращает рейтинг фильма
- **Метод `getName()`**: возвращает название фильма

#### `Course` (Курс)

- **Поля**:
    - `String title` - название курса
    - `String instructor` - преподаватель
    - `double rating` - рейтинг
    - `int enrolledStudents` - количество записанных студентов (опционально)
    - `int duration` - продолжительность в часах (опционально)
- **Конструктор**: принимает все или основные поля
- **Метод `getRating()`**: возвращает рейтинг курса
- **Метод `getName()`**: возвращает название курса

### 3. Класс `RatingService`

Предоставляет сервисные методы для работы с рейтингами:

- **Методы**:
    - `List<Ratable> filterTopRated(List<Ratable> items, int limit)` - возвращает топ N элементов по рейтингу
    - `List<Ratable> filterTopRated(List<Ratable> items, double minRating)` - возвращает элементы с рейтингом выше
      указанного
    - `List<Ratable> filterTopRated(List<Ratable> items, int limit, double minRating)` - комбинированная фильтрация
    - `double getAverageRating(List<Ratable> items)` - средний рейтинг по списку
    - `Ratable findHighestRated(List<Ratable> items)` - находит элемент с максимальным рейтингом
    - `Map<Class, List<Ratable>> groupByType(List<Ratable> items)` - группирует по типу (опционально)
    - `void printTopRated(List<Ratable> items, int limit)` - выводит топ элементов

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт список различных объектов (книги, фильмы, курсы)
- Показывает, что все они реализуют `Ratable`
- Применяет различные методы фильтрации
- Выводит лучшие элементы разных типов
- Демонстрирует полиморфное поведение

## Технические требования

### Полиморфизм через интерфейс

- Все объекты реализуют единый интерфейс `Ratable`
- `RatingService` работает с типом `Ratable`, а не с конкретными классами
- Метод `filterTopRated` может принимать любые объекты, реализующие интерфейс

### Stream API

- Использование Stream API для фильтрации и сортировки
- `sorted()` с компаратором по рейтингу
- `limit()` для ограничения количества
- `filter()` для отбора по минимальному рейтингу

### Инкапсуляция

- Каждый класс инкапсулирует свои специфические поля
- Общее поведение вынесено в интерфейс

### Обработка данных

- Рейтинги могут быть в разных диапазонах (0-5, 0-10)
- Методы фильтрации должны корректно работать с любыми значениями

## Ключевые концепции

- **Полиморфизм**: единый интерфейс для разных типов объектов
- **Интерфейс как контракт**: общее поведение для разных классов
- **Stream API**: функциональная обработка коллекций
- **Слабая связанность**: сервис не зависит от конкретных реализаций
