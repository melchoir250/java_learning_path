# Задача 36: Пользовательский ввод с валидацией (Инкапсуляция)

## Описание проекта

Разработайте систему для безопасного чтения пользовательского ввода, демонстрирующую принципы инкапсуляции и защиты
данных. Класс `UserInput` предоставляет методы для чтения различных типов данных (email, телефон) со встроенной
валидацией. Некорректные данные не сохраняются, а пользователю предлагается ввести их повторно до получения корректного
значения.

## Структура проекта

```
user_input/
├── UserInput.java            # Класс для валидированного ввода
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `UserInput`

Представляет безопасный ввод данных с валидацией:

**Поля**:

- `Scanner scanner` - сканер для чтения ввода (приватное)

**Конструктор**:

- Инициализирует Scanner (например, `new Scanner(System.in)`)

**Методы**:

#### `String readEmail()`

- Читает email от пользователя
- Валидация: проверяет корректность формата email
- При некорректном вводе выводит сообщение об ошибке и предлагает ввести снова
- Возвращает корректный email

#### `String readPhone()`

- Читает номер телефона от пользователя
- Валидация: проверяет формат номера (например, +7XXXXXXXXXX или 8XXXXXXXXXX)
- При некорректном вводе выводит сообщение об ошибке и предлагает ввести снова
- Возвращает корректный номер телефона

### 2. Класс `Main`

Демонстрирует работу системы:

- Создаёт объект `UserInput`
- Запрашивает email и номер телефона
- Показывает, как система обрабатывает некорректный ввод
- Выводит корректные данные после успешной валидации

## Технические требования

### Инкапсуляция

- Scanner и логика валидации скрыты внутри класса
- Внешний код не имеет доступа к сырому вводу

### Валидация

- Встроенные правила для email и телефона
- Возможность расширения правил

### Обработка ошибок

- Информативные сообщения об ошибках
- Повторный запрос до получения корректных данных

### Регулярные выражения

- Использование регулярных выражений для проверки форматов

## Ключевые концепции

- **Инкапсуляция**: скрытие логики валидации
- **Защита от некорректных данных**: несохранение ошибочных значений
- **Повторный запрос**: цикл до получения корректных данных
- **Регулярные выражения**: проверка форматов
