# Задача 35: История действий пользователя (Композиция + Инкапсуляция)

## Описание проекта

Разработайте систему для отслеживания истории действий пользователя, демонстрирующую принципы композиции и инкапсуляции.
Класс `UserActionHistory` содержит список действий, но прямой доступ к этому списку закрыт. Все операции с историей
выполняются через специальные методы, что позволяет контролировать доступ и обеспечивать целостность данных.

## Структура проекта

```
user_action_history/
├── UserActionHistory.java    # Класс истории действий
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `UserActionHistory`

Представляет историю действий пользователя:

**Поля (все приватные)**:

- `List<UserAction> actions` - список действий
- `int maxHistorySize` - максимальный размер истории (опционально)

**Конструктор**:

- Инициализирует пустой список действий
- Может принимать максимальный размер истории

**Методы**:

#### `void addAction(UserAction action)`

- Добавляет новое действие в историю
- Если задан максимальный размер, удаляет самые старые действия при превышении
- Проверяет, что действие не null

#### `UserAction getLastAction()`

- Возвращает последнее действие
- Если история пуста, возвращает null или выбрасывает исключение

#### `UserAction getActionAt(int index)`

- Возвращает действие по индексу (с начала или с конца)
- Проверяет корректность индекса

#### `List<UserAction> getRecentActions(int count)`

- Возвращает список последних N действий
- Возвращает копию списка, а не оригинал

#### `List<UserAction> getAllActions()`

- Возвращает копию всех действий (защита от изменений)

#### `int getActionCount()`

- Возвращает количество действий в истории

#### `void clear()`

- Очищает всю историю

#### `boolean isEmpty()`

- Проверяет, пуста ли история

#### `void undo()`

- Отменяет последнее действие (опционально)
- Удаляет последнее действие из истории

### 2. Класс `UserAction` (Вспомогательный)

Представляет одно действие пользователя:

- **Поля**:
    - `String type` - тип действия (например, "LOGIN", "CLICK", "SEARCH")
    - `String description` - описание действия
    - `LocalDateTime timestamp` - время совершения действия
    - `String userId` - идентификатор пользователя (опционально)
- **Конструктор**: принимает тип и описание
- **Геттеры**: для доступа к полям
- **Метод `toString()`**: для удобного вывода

### 3. Класс `Main`

Демонстрирует работу системы:

- Создаёт историю действий
- Добавляет несколько действий
- Показывает последнее действие
- Показывает последние N действий
- Демонстрирует, что прямой доступ к списку невозможен
- Показывает защиту данных через возврат копий

## Технические требования

### Инкапсуляция

- Список действий должен быть приватным
- Нет прямого доступа к списку извне
- Все методы возвращают копии или защищённые представления

### Композиция

- `UserActionHistory` содержит и управляет списком действий
- Класс полностью контролирует жизненный цикл коллекции

### Управление коллекцией

- Добавление: проверка на null, ограничение размера
- Получение: возврат копий для защиты от изменений
- Удаление: безопасное удаление с проверками

### Потокобезопасность (опционально)

- Использование `synchronized` для многопоточных сценариев
- Или использование `CopyOnWriteArrayList`

## Ключевые концепции

- **Инкапсуляция**: скрытие внутренней структуры данных
- **Композиция**: управление коллекцией через методы
- **Защита данных**: возврат копий для предотвращения несанкционированных изменений
- **Контроль доступа**: все операции через чётко определённый API
