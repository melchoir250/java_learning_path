# Задача 32: Стратегия доставки (Полиморфизм)

## Описание проекта

Разработайте систему расчёта доставки, демонстрирующую паттерн "Стратегия" (Strategy). Различные способы доставки (
курьер, самовывоз, дрон) реализуют единый интерфейс `DeliveryStrategy`. Клиент (сервис доставки) может выбирать
стратегию во время выполнения, что позволяет гибко изменять логику расчёта времени доставки без изменения основного
кода.

## Структура проекта

```
delivery_strategy/
├── DeliveryStrategy.java     # Интерфейс стратегии доставки
├── Courier.java              # Стратегия: доставка курьером
├── Pickup.java               # Стратегия: самовывоз
├── Drone.java                # Стратегия: доставка дроном
├── DeliveryService.java      # Сервис доставки (клиент стратегии)
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `DeliveryStrategy`

Определяет контракт для всех стратегий доставки:

- **Метод `double calculateTime(double distance);`** - рассчитывает время доставки
- **Метод `String getStrategyName();`** - возвращает название стратегии

### 2. Класс `Location` (Вспомогательный)

Представляет географическую точку:

- **Поля**:
    - `double latitude` - широта
    - `double longitude` - долгота
    - `String address` - текстовый адрес (опционально)
- **Методы**:
    - `double distanceTo(Location other)` - рассчитывает расстояние до другой точки

### 3. Реализации стратегий

#### `Courier` (Доставка курьером)

- **Особенности**:
    - Учитывает расстояние и дорожную инфраструктуру
    - Время зависит от пробок (можно добавить фактор загруженности)
    - Работает в определённых зонах доставки
- **Логика `calculateTime()`**:
    - `время = расстояние / средняя скорость курьера (например, 30 км/ч)`
    - Может добавлять время на сборку заказа (30-60 минут)

#### `Pickup` (Самовывоз)

- **Особенности**:
    - Не зависит от расстояния
    - Время фиксированное (готовность заказа)
- **Логика `calculateTime()`**:
    - Фиксированное время (например, 15-30 минут)
    - Не зависит от местоположения клиента

#### `Drone` (Доставка дроном)

- **Особенности**:
    - Высокая скорость, но ограниченная дальность
    - Может быть ограничение по весу
    - Учитывает погодные условия
- **Логика `calculateTime()`**:
    - `время = расстояние / скорость дрона (например, 50 км/ч)`
    - Добавляет время на взлёт и посадку
    - Проверяет дальность действия

### 4. Класс `DeliveryService`

Представляет сервис доставки (клиент стратегии):

- **Поле**:
    - `DeliveryStrategy strategy` - текущая стратегия доставки
- **Конструктор**: принимает начальную стратегию
- **Методы**:
    - `void setStrategy(DeliveryStrategy strategy)` - смена стратегии
    - `double calculateDeliveryTime(Location from, Location to)` - расчёт времени с текущей стратегией
    - `void deliver(Order order, Location destination)` - выполнить доставку
    - `String getCurrentStrategyInfo()` - информация о текущей стратегии

### 5. Класс `Order` (Вспомогательный)

Представляет заказ:

- **Поля**:
    - `String orderId` - идентификатор заказа
    - `List<Product> products` - список товаров
    - `double weight` - общий вес (для ограничений дрона)
- **Методы**: геттеры и сеттеры

### 6. Класс `Main`

Демонстрирует работу системы:

- Создаёт различные стратегии доставки
- Создаёт сервис доставки
- Показывает расчёт времени для разных стратегий
- Демонстрирует смену стратегии во время выполнения
- Показывает, как одна и та же задача решается по-разному
