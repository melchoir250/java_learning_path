# Задача 29: Сортировка объектов (Comparable + Полиморфизм)

## Описание проекта

Разработайте систему для сортировки товаров, демонстрирующую использование встроенного интерфейса Java `Comparable`.
Класс товаров реализует естественный порядок сортировки: сначала по цене, затем по названию. Это позволяет использовать
стандартные механизмы сортировки Java и демонстрирует полиморфизм через интерфейс Java API.

## Структура проекта

```
comparable_product/
├── Product.java              # Класс товара с реализацией Comparable
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `Product`

Представляет товар с возможностью сравнения:

- **Поля** (все приватные):
    - `String id` - уникальный идентификатор товара
    - `String name` - название товара
    - `double price` - цена товара
    - `String category` - категория товара (опционально)
    - `int rating` - рейтинг товара (опционально)
- **Конструктор**: для инициализации всех обязательных полей
- **Геттеры**: для доступа к полям
- **Метод `toString()`**: для удобного вывода информации о товаре

### 2. Реализация `Comparable<Product>`

Класс `Product` должен реализовывать интерфейс `Comparable<Product>`:

- **Метод `int compareTo(Product other)`**:
    - Сравнивает текущий товар с другим товаром
    - Возвращает отрицательное число, если текущий товар "меньше"
    - Возвращает ноль, если товары равны
    - Возвращает положительное число, если текущий товар "больше"
- **Правила сравнения**:
    1. Сначала сравнение по цене (от меньшей к большей)
    2. Если цены равны, сравнение по названию (в алфавитном порядке)
    3. Если и названия равны, сравнение по ID (опционально)

### 3. Класс `Main`

Демонстрирует работу системы:

- Создаёт список товаров в произвольном порядке
- Выводит исходный список
- Сортирует список с помощью `Collections.sort()` или `List.sort()`
- Выводит отсортированный список
- Показывает, что сортировка работает автоматически благодаря реализации `Comparable`
- Демонстрирует сравнение отдельных товаров

## Технические требования

### Реализация Comparable

- Корректная реализация контракта `compareTo()`
- Соблюдение рефлексивности: `x.compareTo(x) == 0`
- Симметричности: `x.compareTo(y) == -y.compareTo(x)`
- Транзитивности: если `x.compareTo(y) > 0` и `y.compareTo(z) > 0`, то `x.compareTo(z) > 0`
- Согласованность с `equals()` (рекомендуется)

### Полиморфизм

- `Collections.sort()` работает с любым списком объектов, реализующих `Comparable`
- Стандартные методы Java автоматически используют нашу реализацию

### Сравнение

- Использование `Double.compare()` для сравнения цен (обработка NaN)
- Использование `String.compareTo()` для сравнения названий

## Ключевые концепции

- **Comparable**: встроенный интерфейс Java для определения естественного порядка
- **Полиморфизм**: стандартные методы Java работают с нашим классом
- **Контракт compareTo()**: математические свойства сравнения
- **Естественный порядок**: логика сортировки по умолчанию
