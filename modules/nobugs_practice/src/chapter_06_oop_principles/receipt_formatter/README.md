# Задача 31: Чек и форматирование (Абстракция + Шаблон)

## Описание проекта

Разработайте систему для формирования чеков, демонстрирующую принцип разделения данных и представления (MVC).
Абстрактный класс `Receipt` определяет структуру данных чека и предоставляет абстрактный метод `format()` для
форматирования. Различные реализации форматирования позволяют выводить чек в разных форматах (текст, HTML, JSON) без
изменения логики работы с данными.

## Структура проекта

```
receipt_formatter/
├── Receipt.java              # Абстрактный класс чека
├── PlainTextReceipt.java     # Форматирование в простой текст
├── HtmlReceipt.java          # Форматирование в HTML
├── JsonReceipt.java          # Форматирование в JSON
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Класс `Receipt` (Абстрактный)

Представляет структуру чека и определяет контракт форматирования:

- **Поля**:
    - `String storeName` - название магазина
    - `String storeAddress` - адрес магазина (опционально)
    - `LocalDateTime date` - дата и время покупки
    - `List<ReceiptItem> items` - список позиций чека
    - `double tax` - налог (в процентах или сумме)
    - `double discount` - скидка (опционально)
- **Конструктор**: принимает название магазина, дату и список позиций
- **Методы** (конкретные):
    - `double calculateSubtotal()` - расчёт суммы без налога и скидки
    - `double calculateTaxAmount()` - расчёт суммы налога
    - `double calculateTotal()` - расчёт итоговой суммы
    - `void addItem(ReceiptItem item)` - добавить позицию
    - `void removeItem(String productName)` - удалить позицию
- **Абстрактный метод**:
    - `String format()` - возвращает отформатированный чек

### 2. Класс `ReceiptItem` (Вспомогательный)

Представляет позицию в чеке:

- **Поля**:
    - `String name` - название товара
    - `int quantity` - количество
    - `double unitPrice` - цена за единицу
- **Методы**:
    - `double getTotal()` - возвращает общую стоимость позиции
    - Геттеры для всех полей

### 3. Реализации форматирования

#### `PlainTextReceipt`

- Форматирует чек в виде простого текста
- **Структура**:
  ```
  ======================
     НАЗВАНИЕ МАГАЗИНА
     Адрес магазина
  ======================
  Дата: 15.03.2024 14:30
  
  -------------------------------------
  Товар          Кол-во    Цена    Сумма
  -------------------------------------
  Хлеб           2         50.00   100.00
  Молоко         1         80.00   80.00
  -------------------------------------
  Подытог:                   180.00
  Налог (10%):                18.00
  -------------------------------------
  ИТОГО:                      198.00
  ======================
  Спасибо за покупку!
  ```

#### `HtmlReceipt`

- Форматирует чек в HTML
- **Структура**:
  ```html
  <div class="receipt">
    <h1>НАЗВАНИЕ МАГАЗИНА</h1>
    <p>Адрес магазина</p>
    <p>Дата: 15.03.2024 14:30</p>
    <table>
      <thead>
        <th>Товар</th><th>Кол-во</th><th>Цена</th><th>Сумма</th></tr>
      </thead>
      <tbody>
        <tr><td>Хлеб</td><td>2</td><td>50.00</td><td>100.00</td></tr>
        <tr><td>Молоко</td><td>1</td><td>80.00</td><td>80.00</td></tr>
      </tbody>
    </table>
    <p>Подытог: 180.00</p>
    <p>Налог (10%): 18.00</p>
    <p><strong>ИТОГО: 198.00</strong></p>
    <p>Спасибо за покупку!</p>
  </div>
  ```

#### `JsonReceipt`

- Форматирует чек в JSON
- **Структура**:
  ```json
  {
    "storeName": "НАЗВАНИЕ МАГАЗИНА",
    "storeAddress": "Адрес магазина",
    "date": "2024-03-15T14:30:00",
    "items": [
      {
        "name": "Хлеб",
        "quantity": 2,
        "unitPrice": 50.00,
        "total": 100.00
      },
      {
        "name": "Молоко",
        "quantity": 1,
        "unitPrice": 80.00,
        "total": 80.00
      }
    ],
    "subtotal": 180.00,
    "tax": 18.00,
    "total": 198.00,
    "message": "Спасибо за покупку!"
  }
  ```

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт чек с данными (позиции, налог, скидка)
- Показывает чек в разных форматах
- Демонстрирует, что данные чека не зависят от способа форматирования
- Показывает лёгкость добавления новых форматов

## Технические требования

### Абстракция

- `Receipt` содержит общую логику расчётов
- Абстрактный метод `format()` определяет контракт для форматирования
- Конкретные реализации отвечают только за представление

### Разделение ответственности

- Данные и бизнес-логика в абстрактном классе
- Представление вынесено в реализации
- Легко добавить новый формат без изменения существующего кода

### Шаблонный метод (Template Method)

- `Receipt` может содержать шаблонные методы, использующие `format()`
- Общая структура форматирования может быть задана в базовом классе

## Ключевые концепции

- **Абстракция**: выделение общего поведения в абстрактный класс
- **Разделение данных и представления**: данные не зависят от формата вывода
- **Шаблонный метод**: алгоритм форматирования может быть определён в базовом классе
- **Расширяемость**: новые форматы добавляются без изменения существующего кода
