# Задача 23: Расписание транспорта (Абстракция + Расширение)

## Описание проекта

Разработайте систему расписания транспорта, демонстрирующую принципы абстракции и расширяемости через наследование.
Абстрактный класс определяет общий контракт для всех видов транспорта, а конкретные реализации предоставляют свою логику
получения расписания. Это позволяет легко добавлять новые виды транспорта без изменения существующего кода.

## Структура проекта

```
transport_schedule/
├── TransportSchedule.java    # Абстрактный класс расписания
├── BusSchedule.java          # Расписание автобусов
├── TrainSchedule.java        # Расписание поездов
├── FlightSchedule.java       # Расписание рейсов
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Абстрактный класс `TransportSchedule`

Определяет общий контракт для всех видов расписания:

- **Абстрактные методы**:
    - `String getNextDeparture()` - возвращает время следующего отправления
    - `String getSchedule()` - возвращает полное расписание (опционально)
    - `boolean isAvailable(String time)` - проверяет наличие рейса в указанное время (опционально)
- **Конкретные методы**:
    - `void displayNextDeparture()` - выводит следующее отправление (использует абстрактный метод)
    - `String getTransportType()` - возвращает тип транспорта
- **Поля**:
    - `String route` - маршрут (например, "Москва - Санкт-Петербург")
    - `String transportNumber` - номер рейса/маршрута

### 2. Реализации расписаний

#### `BusSchedule` (Расписание автобусов)

- **Особенности**:
    - Автобусы ходят с определённой периодичностью (например, каждые 30 минут)
    - Может учитывать время работы (с 6:00 до 23:00)
    - Может иметь разные интервалы в разное время дня
- **Логика метода `getNextDeparture()`**:
    - Рассчитывает ближайшее время отправления от текущего момента
    - Учитывает расписание движения

#### `TrainSchedule` (Расписание поездов)

- **Особенности**:
    - Поезда отправляются по фиксированному расписанию (например, в 8:00, 12:30, 18:45)
    - Может иметь разные маршруты и типы поездов (скорый, пассажирский)
    - Может учитывать дни недели
- **Логика метода `getNextDeparture()`**:
    - Находит ближайший рейс из фиксированного списка
    - Учитывает текущее время

#### `FlightSchedule` (Расписание рейсов)

- **Особенности**:
    - Рейсы выполняются по дням недели (например, понедельник, среда, пятница)
    - Может учитывать сезонность
    - Может иметь разные терминалы
- **Логика метода `getNextDeparture()`**:
    - Определяет ближайший рейс с учётом дней недели
    - Может учитывать часовые пояса

### 3. Класс `Main`

Демонстрирует работу системы:

- Создаёт объекты различных расписаний
- Для каждого вызывает метод `getNextDeparture()`
- Показывает полиморфное поведение
- Демонстрирует, что все объекты работают через единый интерфейс абстрактного класса

## Технические требования

### Абстракция

- `TransportSchedule` содержит общие поля и методы для всех видов транспорта
- Абстрактные методы определяют контракт, который должны реализовать подклассы
- Конкретные методы могут быть переопределены при необходимости

### Наследование

- Все конкретные расписания наследуют от `TransportSchedule`
- Каждый подкласс реализует абстрактные методы по-своему
- Подклассы могут добавлять свои специфические поля и методы

### Полиморфизм

- Объекты разных типов могут храниться в коллекции `TransportSchedule`
- Вызов `getNextDeparture()` работает по-разному в зависимости от реального типа

### Расширяемость

- Новый вид транспорта (например, `ShipSchedule`) можно добавить без изменения существующих классов
- Достаточно создать новый подкласс и реализовать абстрактные методы

## Ключевые концепции

- **Абстрактный класс**: общий контейнер для иерархии классов
- **Абстрактные методы**: определение контракта без реализации
- **Наследование**: повторное использование кода и расширение функциональности
- **Полиморфизм**: единый интерфейс для разных реализаций

## Дополнительные требования (опционально)

1. **Работа с временем**:
    - Использование `LocalTime` или `LocalDateTime` для представления времени
    - Учёт текущего момента при расчёте следующего отправления

2. **Разные типы расписания**:
    - `FixedSchedule` - фиксированные времена отправления
    - `IntervalSchedule` - регулярные интервалы
    - `WeeklySchedule` - расписание по дням недели

3. **Задержки и отмены**:
    - Добавить метод `isDelayed()` или `isCancelled()`
    - Учитывать статус рейса при расчёте

4. **Фильтрация**:
    - Метод для получения расписания на определённый день
    - Фильтрация по времени отправления

5. **Сравнение**:
    - Метод для сравнения времени до следующего отправления между разными видами транспорта

6. **Файловое хранение**:
    - Загрузка расписания из файла
    - Сохранение изменений

## Критерии оценки реализации

1. **Абстрактный класс**:
    - Содержит общие поля и методы для всех подклассов
    - Имеет хотя бы один абстрактный метод

2. **Наследование**:
    - Подклассы корректно наследуют и расширяют функциональность
    - Реализуют все абстрактные методы

3. **Полиморфизм**:
    - Код в `Main` работает с типом `TransportSchedule`
    - Вызов одного и того же метода даёт разный результат

4. **Расширяемость**:
    - Можно легко добавить новый вид транспорта
    - Существующий код не требует изменений

## Как должен работать код

### Этап 1: Создание расписаний

1. Создаются объекты `BusSchedule`, `TrainSchedule`, `FlightSchedule`
2. Каждый объект инициализируется с параметрами маршрута и расписания

### Этап 2: Получение информации

1. Для каждого объекта вызывается метод `getNextDeparture()`
2. Каждый объект возвращает время следующего отправления в своём формате
3. Метод `displayNextDeparture()` выводит информацию

### Этап 3: Полиморфная обработка

1. Все объекты сохраняются в список типа `List<TransportSchedule>`
2. Цикл проходит по списку и вызывает одинаковые методы
3. Каждый объект выполняет свою специфическую логику

## Добавление нового вида транспорта (Расширение)

```
=== Добавляем новый класс ShipSchedule ===
public class ShipSchedule extends TransportSchedule {
    // Реализация методов для парома
}

=== Создаём расписание парома ===
Маршрут: Санкт-Петербург - Хельсинки
Расписание: ежедневно в 10:00, 14:00, 18:00
Текущее время: 13:30
Следующее отправление: 14:00

=== Система работает без изменений ===
Новый тип автоматически работает со всем существующим кодом!
```

## Принципы, демонстрируемые задачей

1. **Абстракция**: выделение общего поведения в абстрактный класс
2. **Наследование**: переиспользование кода и расширение функциональности
3. **Полиморфизм**: единый интерфейс для разных реализаций
4. **Открытость/закрытость**: система открыта для расширения (новые виды транспорта), но закрыта для модификации (
   существующий код не меняется)
