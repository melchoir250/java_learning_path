# Задача 20: Устройства и управление (Композиция + Полиморфизм)

## Описание проекта

Разработайте систему управления умными устройствами, демонстрирующую принципы композиции и полиморфизма. Различные
устройства реализуют единый интерфейс включения/выключения, а пульт дистанционного управления может работать с любым
таким устройством через композицию, а не наследование.

## Структура проекта

```
smart_devices/
├── Switchable.java           # Интерфейс управляемого устройства
├── Lamp.java                 # Реализация лампы
├── Fan.java                  # Реализация вентилятора
├── TV.java                   # Реализация телевизора
├── RemoteController.java     # Пульт управления (композиция)
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `Switchable`

Определяет контракт для всех управляемых устройств:

- **Метод `void turnOn()`** - включает устройство
- **Метод `void turnOff()`** - выключает устройство
- **Метод `boolean isOn()`** - возвращает текущее состояние (опционально)
- **Метод `String getStatus()`** - возвращает строковое представление состояния (опционально)

### 2. Реализации устройств

#### `Lamp` (Лампа)

- **Поля**:
    - `String location` - место расположения (например, "гостиная", "спальня")
    - `int brightness` - текущая яркость (0-100)
    - `boolean isOn` - состояние включения
- **Особенности**:
    - При включении устанавливает яркость на последнее значение или значение по умолчанию
    - Можно добавить метод `setBrightness(int level)` для регулировки яркости

#### `Fan` (Вентилятор)

- **Поля**:
    - `String location` - место расположения
    - `int speed` - текущая скорость (0-5, где 0 - выключен)
    - `boolean isOn` - состояние включения
- **Особенности**:
    - При включении устанавливает скорость на последнее значение или 1
    - Можно добавить метод `setSpeed(int speed)` для регулировки скорости

#### `TV` (Телевизор)

- **Поля**:
    - `String location` - место расположения
    - `int channel` - текущий канал
    - `int volume` - текущая громкость
    - `boolean isOn` - состояние включения
- **Особенности**:
    - При включении возвращается на последний канал и громкость
    - Можно добавить методы `setChannel(int channel)`, `setVolume(int volume)`

### 3. Класс `RemoteController`

Представляет пульт дистанционного управления:

- **Поле**:
    - `Switchable device` - устройство, которым управляет пульт (композиция)
- **Конструктор**: принимает любое устройство, реализующее `Switchable`
- **Методы**:
    - `void pressPowerButton()` - включает/выключает устройство
    - `void setDevice(Switchable device)` - возможность сменить устройство
    - `Switchable getDevice()` - получить текущее устройство
    - `void showStatus()` - выводит текущее состояние устройства

### 4. Класс `Main`

Демонстрирует работу системы:

- Создаёт различные устройства (Lamp, Fan, TV)
- Создаёт пульт управления
- Показывает, как один пульт может управлять разными устройствами
- Демонстрирует смену устройства
- Выводит состояния устройств до и после управления

## Технические требования

### Композиция вместо наследования

- `RemoteController` содержит ссылку на `Switchable` (has-a отношение)
- Не используется наследование для расширения функциональности
- Пульт может работать с любым устройством, реализующим интерфейс

### Полиморфизм

- Все устройства реализуют единый интерфейс `Switchable`
- Методы пульта работают с интерфейсом, а не с конкретными классами
- Один и тот же пульт может управлять лампой, вентилятором и телевизором

### Инкапсуляция

- Состояние каждого устройства скрыто внутри класса
- Управление состоянием происходит только через методы интерфейса

## Ключевые концепции

- **Композиция**: `RemoteController` содержит ссылку на устройство, а не наследует его поведение
- **Полиморфизм**: единый интерфейс для разных типов устройств
- **Слабая связанность**: пульт зависит только от интерфейса, а не от конкретных реализаций
- **Инкапсуляция**: внутреннее состояние устройств скрыто

## Дополнительные требования (опционально)

1. **Многофункциональный пульт**:
    - Добавить кнопки для регулировки яркости/скорости/громкости
    - Добавить кнопки для переключения каналов

2. **Универсальный пульт**:
    - Возможность управления несколькими устройствами одновременно
    - Хранение нескольких устройств с возможностью выбора

3. **Макросы**:
    - Создание сценариев (например, "режим кино": выключить свет, включить TV, установить громкость 20)

4. **Состояние устройств**:
    - Сохранение состояния устройств при выключении
    - Восстановление последнего состояния при включении

5. **Групповое управление**:
    - Класс `DeviceGroup`, реализующий `Switchable` и управляющий группой устройств
    - Включение/выключение всех устройств в группе

6. **Расписание**:
    - Возможность запланировать включение/выключение устройств

## Критерии оценки реализации

1. **Композиция**:
    - `RemoteController` содержит ссылку на устройство
    - Нет наследования между пультом и устройствами

2. **Полиморфизм**:
    - Все устройства реализуют `Switchable`
    - Пульт работает с типом `Switchable`

3. **Интерфейс**:
    - Чётко определённый контракт для управляемых устройств
    - Все устройства корректно реализуют методы включения/выключения

4. **Гибкость**:
    - Возможность смены устройства без создания нового пульта
    - Пульт может работать с любым будущим устройством, реализующим интерфейс

## Как должен работать код

### Этап 1: Создание устройств

1. Создаются объекты различных устройств (Lamp, Fan, TV)
2. Каждое устройство инициализируется с определёнными параметрами

### Этап 2: Управление через пульт

1. Создаётся пульт, который связывается с одним устройством
2. Пульт управляет устройством через общие методы `turnOn()` и `turnOff()`
3. Устройство выполняет свою специфическую логику включения/выключения

### Этап 3: Смена устройства

1. Пульт может быть перенастроен на другое устройство
2. Те же самые кнопки управляют новым устройством
3. Код управления не меняется

## Пример работы (логика)

```
=== Создаём устройства ===
Лампа в гостиной (выключена)
Вентилятор в спальне (выключен, скорость 0)
Телевизор в гостиной (выключен, канал 1, громкость 10)

=== Пульт управляет лампой ===
Нажимаем кнопку питания...
Лампа в гостиной включена (яркость: 80%)

=== Меняем устройство на вентилятор ===
Нажимаем кнопку питания...
Вентилятор в спальне включен (скорость: 3)

=== Меняем устройство на телевизор ===
Нажимаем кнопку питания...
Телевизор в гостиной включен (канал: 5, громкость: 15)

=== Выключаем все устройства ===
Телевизор выключен
Вентилятор выключен
Лампа выключена
```

## Принципы, демонстрируемые задачей

1. **Композиция вместо наследования**: пульт "имеет" устройство, а не "является" устройством
2. **Полиморфизм через интерфейс**: единый способ управления разными устройствами
3. **Слабая связанность**: пульт не знает о конкретных типах устройств
4. **Гибкость и расширяемость**: новые устройства можно добавлять без изменения пульта
