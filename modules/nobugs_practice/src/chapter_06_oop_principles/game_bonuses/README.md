# Задача 17: Бонусы в игре (Полиморфизм через интерфейс)

## Описание проекта
Разработайте систему игровых бонусов, демонстрирующую принцип полиморфизма через интерфейсы. Система должна позволять применять различные типы бонусов к игроку, причём каждый бонус реализует единый интерфейс и может быть легко заменён или расширен.

## Структура проекта

```
game_bonuses/
├── BonusEffect.java          # Интерфейс эффекта бонуса
├── HealthBoost.java          # Бонус к здоровью
├── SpeedBoost.java           # Бонус к скорости
├── DamageMultiplier.java     # Множитель урона
├── Player.java               # Класс игрока
├── Main.java                 # Демонстрация работы
└── README.md                 # Описание задачи
```

## Функциональные требования

### 1. Интерфейс `BonusEffect`
- Определяет контракт для всех игровых бонусов
- Содержит один метод `void applyTo(Player player)`
- Каждая реализация будет по-своему изменять характеристики игрока

### 2. Класс `Player`
Представляет игрока с набором характеристик:
- **Поля**:
    - `String name` - имя игрока
    - `double health` - текущее здоровье (или максимальное здоровье)
    - `double speed` - скорость передвижения
    - `double damage` - наносимый урон
    - дополнительные характеристики по вашему усмотрению
- **Конструктор**: для инициализации имени и базовых характеристик
- **Геттеры и сеттеры**: для доступа к характеристикам
- **Метод `applyBonus(BonusEffect bonus)`**: принимает любой бонус и применяет его к игроку
- **Метод `getStats()`**: возвращает строку с текущими характеристиками
- **Метод `resetToBase()`**: сбрасывает характеристики к базовым значениям (опционально)

### 3. Реализации бонусов

#### `HealthBoost`
- Увеличивает здоровье игрока на фиксированное значение (например, +20)
- Или увеличивает максимальное здоровье на процент (например, +25%)
- Выберите подходящий вариант реализации

#### `SpeedBoost`
- Увеличивает скорость игрока на фиксированное значение (например, +5.0)
- Или увеличивает скорость на процент (например, +30%)
- Выберите подходящий вариант реализации

#### `DamageMultiplier`
- Увеличивает наносимый урон в заданное количество раз (например, x2)
- Принимает множитель в конструкторе (например, 2.0 для двойного урона)
- Может быть реализован как временный или постоянный эффект

### 4. Класс `Main`
- Демонстрирует работу системы:
    - Создаёт игрока с базовыми характеристиками
    - Выводит начальные характеристики
    - Создаёт список бонусов (разные типы)
    - Последовательно применяет бонусы к игроку
    - После каждого бонуса выводит изменённые характеристики
    - Показывает, что все бонусы работают через единый интерфейс

## Технические требования

### Полиморфизм
- Все бонусы реализуют единый интерфейс `BonusEffect`
- Метод `applyBonus` в классе `Player` принимает параметр типа интерфейс
- В `Main` бонусы могут храниться в списке типа `List<BonusEffect>`

### Инкапсуляция
- Характеристики игрока должны быть скрыты (private)
- Изменение характеристик происходит только через методы

### Гибкость
- Новые типы бонусов можно добавлять без изменения существующего кода
- Бонусы могут быть скомбинированы в любом порядке

## Ключевые концепции
- Полиморфизм через интерфейсы
- Единый контракт для разных поведений
- Инкапсуляция характеристик игрока
- Открытость для расширения, закрытость для модификации (принцип Open-Closed)
