# Задача 6: StackOverflowError в рекурсии

## Описание
Программа должна напечатать числа от n до 1, используя рекурсивный метод, но при запуске падает с ошибкой `StackOverflowError`. Ваша задача — с помощью отладчика найти причину ошибки и исправить код.

## Условие
```java
public class DebugTask6 {
    public static void main(String[] args) {
        countdown(5);
    }
    
    public static void countdown(int n) {
        System.out.println(n);
        countdown(n - 1);
    }
}
```

## Ожидаемый результат
Программа должна вывести числа от 5 до 1:
```
5
4
3
2
1
```

## Фактическое поведение
При запуске программа выводит несколько чисел, а затем выбрасывает исключение:
```
5
4
3
2
1
0
-1
-2
...
Exception in thread "main" java.lang.StackOverflowError
    at java.base/java.io.PrintStream.println(PrintStream.java:877)
    at DebugTask6.countdown(DebugTask6.java:7)
    at DebugTask6.countdown(DebugTask6.java:8)
    ...
```

## Задание
1. Запустите программу в режиме отладки (Debug).
2. Установите точку останова (breakpoint) в начале метода `countdown`.
3. Проанализируйте значения переменной `n` на каждом шаге выполнения.
4. Проследите, сколько раз вызывается метод и когда происходит остановка.
5. Найдите причину бесконечной рекурсии.
6. Исправьте код так, чтобы он работал согласно ожиданию.
7. Запишите процесс отладки и исправления на видео.

## Подсказки для отладки
- Обратите внимание на то, когда должно происходить завершение рекурсии.
- Проверьте, есть ли в методе условие выхода (базовый случай).
- Понаблюдайте за стеком вызовов (call stack) в отладчике.
