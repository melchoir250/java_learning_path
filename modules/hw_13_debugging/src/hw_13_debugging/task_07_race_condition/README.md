# Задача 7: Состояние гонки (Race Condition)

## Описание
Программа моделирует снятие денег со счёта двумя потоками одновременно. Начальный баланс составляет 100 единиц. Первый поток пытается снять 60, второй — 50. Ожидается, что второй поток должен получить отказ, так как после первого снятия остаётся только 40. Однако программа допускает отрицательный баланс. Ваша задача — с помощью отладчика найти причину ошибки и исправить код.

## Условие
```java
public class DebugTask7 {
    private static int balance = 100;
    
    public static void main(String[] args) {
        Thread t1 = new Thread(() -> withdraw(60));
        Thread t2 = new Thread(() -> withdraw(50));
        t1.start();
        t2.start();
    }
    
    public static void withdraw(int amount) {
        if (balance >= amount) {
            try { Thread.sleep(100); } catch (InterruptedException e) { }
            balance -= amount;
            System.out.println("New balance: " + balance);
        }
    }
}
```

## Ожидаемый результат
Программа должна корректно обработать обе операции и не допустить отрицательного баланса. Возможные корректные сценарии:
- Первый поток снимает 60 (остаток 40), второй получает отказ.
- Или (в зависимости от порядка выполнения) второй снимает 50 (остаток 50), первый получает отказ.

Баланс никогда не должен становиться отрицательным.

## Фактическое поведение
При запуске баланс может стать отрицательным, например:
```
New balance: 40
New balance: -10
```
Или:
```
New balance: 50
New balance: -10
```

## Задание
1. Запустите программу в режиме отладки (Debug).
2. Установите точки останова (breakpoints) внутри метода `withdraw`.
3. Запустите оба потока и проанализируйте последовательность выполнения.
4. Обратите внимание на значения переменной `balance` в разные моменты времени.
5. Найдите причину возникновения отрицательного баланса.
6. Исправьте код так, чтобы он работал согласно ожиданию.
7. Запишите процесс отладки и исправления на видео.

## Подсказки для отладки
- Обратите внимание на то, что оба потока могут одновременно войти в блок `if`.
- Проверьте, что происходит с балансом между проверкой условия и фактическим снятием.
- Подумайте, как обеспечить атомарность операции "проверить и снять".
