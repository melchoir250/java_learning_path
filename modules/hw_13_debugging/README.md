# Домашнее задание: Решение проблем (Debugging)

## Цель работы
Научиться находить и исправлять ошибки в коде с помощью отладчика (debugger). В этом задании собраны типичные ошибки, которые допускают разработчики: от простых синтаксических и логических ошибок до проблем с многопоточностью и работой с памятью.

## Что нужно сделать
1. Запустить каждый из предложенных классов.
2. С помощью отладчика (debugger) проанализировать выполнение программы:
    * Установить точки останова (breakpoints).
    * Пройти программу по шагам (step over, step into).
    * Отследить значения переменных на каждом шаге.
3. Найти причину ошибки.
4. Исправить код так, чтобы он работал согласно ожиданию (комментарий в начале каждого класса).
5. Записать видео процесса отладки и исправления для каждой задачи (или одного общего процесса).

## Краткое описание задач

### Задача 1: Выход за границы массива
**Файл:** `task_01_array_print/DebugTask1.java`

Код пытается напечатать все элементы массива, но выбрасывает исключение `ArrayIndexOutOfBoundsException`.

**Ошибка:** В цикле `for` используется условие `i <= numbers.length` вместо `i < numbers.length`.

**Ожидание:** Программа должна напечатать: 10, 20, 30, 40, 50

---

### Задача 2: Бесконечный цикл (неправильное условие)
**Файл:** `task_02_sum_bug/DebugTask2.java`

Код должен вычислить сумму чисел от 1 до 5, но программа либо зависает, либо выдаёт неправильный результат.

**Ошибка:** В цикле `for` используется `i--` вместо `i++`. Счётчик уменьшается, и условие `i <= n` всегда истинно.

**Ожидание:** Сумма 1+2+3+4+5 = 15

---

### Задача 3: Зависание (отсутствует инкремент)
**Файл:** `task_03_infinite_while/DebugTask3.java`

Код должен напечатать числа от 1 до 5, но программа зависает (бесконечный цикл).

**Ошибка:** В цикле `while` отсутствует инкремент переменной `i`. Значение `i` всегда равно 1.

**Ожидание:** Вывод чисел 1, 2, 3, 4, 5

---

### Задача 4: NullPointerException
**Файл:** `task_04_palindrome_null/DebugTask4.java`

Код должен проверить, является ли строка палиндромом, но выбрасывает `NullPointerException`.

**Ошибка:** В метод передаётся `null`, а `StringBuilder` не может работать с `null`.

**Ожидание:** Метод должен корректно обрабатывать `null` (возможно, возвращать `false`).

---

### Задача 5: Изменение не того объекта
**Файл:** `task_05_update_age/DebugTask5.java`

Код должен увеличить возраст человека, но возраст остаётся прежним.

**Ошибка:** В методе `updateAge` создаётся новый объект `Person`, а не изменяется переданный. Ссылка `person` внутри метода перезаписывается, но внешняя ссылка остаётся неизменной.

**Ожидание:** Возраст должен увеличиться на 1.

---

### Задача 6: StackOverflowError (рекурсия без выхода)
**Файл:** `task_06_countdown_recursion/DebugTask6.java`

Код должен напечатать числа от n до 1, но программа падает с `StackOverflowError`.

**Ошибка:** В рекурсивном методе отсутствует базовый случай (условие выхода). Рекурсия продолжается бесконечно.

**Ожидание:** Для n=5 вывод: 5, 4, 3, 2, 1

---

### Задача 7: Состояние гонки (Race condition)
**Файл:** `task_07_race_condition/DebugTask7.java`

Два потока списывают деньги со счёта одновременно. Баланс становится отрицательным.

**Ошибка:** Отсутствует синхронизация доступа к общему ресурсу `balance`. Потоки проверяют условие одновременно, и оба проходят проверку.

**Ожидание:** Баланс не должен стать отрицательным. Второй поток должен получить отказ, так как денег недостаточно.

---

### Задача 8: Ошибка сравнения double
**Файл:** `task_08_double_comparison/DebugTask8.java`

Код сравнивает два числа с плавающей точкой, но результат не соответствует ожиданиям.

**Ошибка:** Из-за особенностей представления чисел с плавающей точкой в памяти, `0.1 * 3` не равно точно `0.3`. Сравнивать `double` через `==` нельзя.

**Ожидание:** Программа должна корректно сравнивать числа с учётом погрешности (epsilon).

---

### Задача 9: Сравнение строк через ==
**Файл:** `task_09_string_comparison/DebugTask9.java`

Код проверяет, равны ли две строки, но всегда выводит "Not Equal".

**Ошибка:** Строки сравниваются через оператор `==`, который сравнивает ссылки, а не содержимое. Хотя строки содержат одинаковый текст, это разные объекты в памяти.

**Ожидание:** Сравнение должно выполняться через метод `equals()`.

---

### Задача 10: ConcurrentModificationException
**Файл:** `task_10_list_remove/DebugTask10.java`

Код пытается удалить элементы списка во время итерации по нему, но выбрасывает `ConcurrentModificationException`.

**Ошибка:** Нельзя модифицировать коллекцию (удалять элементы) во время итерации по ней с помощью for-each или итератора без использования самого итератора.

**Ожидание:** Удалить все имена, начинающиеся на 'A', без исключений.

## Инструкция по выполнению

### Подготовка
1. Откройте проект в вашей IDE (IntelliJ IDEA, Eclipse или другой).
2. Убедитесь, что все файлы скомпилированы без ошибок.

### Процесс отладки для каждой задачи
1. **Запустите программу** и убедитесь, что ошибка воспроизводится.
2. **Установите точку останова (breakpoint)** в начале метода `main` или в месте, где предположительно возникает ошибка.
3. **Запустите программу в режиме отладки** (Debug).
4. **Используйте шаги выполнения:**
    * `Step Over` (F8) — выполнить текущую строку и перейти к следующей.
    * `Step Into` (F7) — войти внутрь вызываемого метода.
    * `Step Out` (Shift+F8) — выйти из текущего метода.
    * `Resume Program` (F9) — продолжить выполнение до следующей точки останова.
5. **Отслеживайте значения переменных** в панели Variables/Debugger.
6. **Когда найдёте причину**, остановите отладку и исправьте код.
7. **Запустите программу снова** (обычный run), чтобы убедиться, что она работает корректно.
8. **Запишите процесс** на видео (можно отдельно для каждой задачи или один ролик, демонстрирующий исправление всех ошибок).
