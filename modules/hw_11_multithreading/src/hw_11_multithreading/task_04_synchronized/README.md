# Задача 4: Использование synchronized

**Уровень сложности:** Средний

## Условие задачи

Напишите класс Counter с методом increment, увеличивающим значение счётчика. Создайте два потока, каждый из которых
вызывает increment() 1000 раз. Обеспечьте правильную работу с помощью synchronized.

## Требования

- Создать класс `Counter` с приватной переменной `count`
- Реализовать метод `increment()`, увеличивающий `count` на 1
- Метод должен быть синхронизирован (использовать `synchronized`)
- Создать два потока, каждый вызывает `increment()` 1000 раз
- После завершения потоков вывести финальное значение счетчика
- Значение должно быть ровно 2000

## Пример вывода

```
Финальное значение счетчика: 2000
```

## Что практикуется

- Проблема состояния гонки (race condition)
- Критическая секция и необходимость синхронизации
- Использование ключевого слова `synchronized` для методов
- Атомарность операций в многопоточной среде

## Подсказки

1. **Race condition**: без синхронизации два потока могут одновременно прочитать одно значение и оба увеличить его, что
   приведет к потере одного инкремента
2. **synchronized метод**: `public synchronized void increment() { count++; }`
3. **Ожидание потоков**: используйте `join()` в main, чтобы дождаться завершения обоих потоков
4. **Эксперимент**: попробуйте убрать `synchronized` и посмотреть, сколько раз получится
