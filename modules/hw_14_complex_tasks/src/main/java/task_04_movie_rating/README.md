# Задача: Система рейтинга фильмов с многопоточной обработкой

## Описание проекта
Разработайте систему для сбора и анализа пользовательских оценок фильмов с поддержкой различных типов рейтингов. Система должна обеспечивать потокобезопасное хранение данных, валидацию пользовательского ввода и предоставлять аналитические возможности с использованием Stream API.

## Функциональные требования

### 1. Класс `Movie`
Создайте класс, представляющий фильм:
- **Поля**:
   - `String title` - название фильма
   - дополнительные характеристики по вашему усмотрению (например, год выпуска, режиссер, жанр)
- **Конструкторы**: для инициализации полей
- **Методы доступа**: стандартные геттеры
- **Методы `equals()` и `hashCode()`**: должны быть корректно реализованы для использования в качестве ключа в `Map`

### 2. Класс `Rating<T extends Number>`
Создайте обобщенный класс для хранения оценки:
- **Поле**: `T value` - значение рейтинга (параметризованный тип, ограниченный Number)
- **Конструктор**: принимает значение рейтинга с валидацией
- **Метод `getValue()`**: возвращает числовое значение
- **Дополнительно**: возможность хранения информации о пользователе, который поставил оценку

### 3. Класс `MovieService`
Реализуйте основной сервис для управления фильмами и оценками:

**Структура данных:**
- Используйте `Map<Movie, List<Rating>>` для хранения всех оценок
- Ключом выступает объект фильма, значением - список оценок

**Методы:**

#### `addRating(Movie movie, Rating rating)`
- Добавляет оценку для указанного фильма
- **Валидация**: проверяет, что оценка находится в диапазоне от 1 до 10 включительно
- **Потокобезопасность**: метод должен быть потокобезопасным (синхронизация на уровне метода или объекта)
- **Обработка ошибок**: выбрасывает исключение при некорректном значении оценки

#### `getAverageRating(Movie movie)`
- Рассчитывает среднюю оценку для конкретного фильма
- Использует Stream API для обработки данных
- Возвращает `OptionalDouble` (или `double` с обработкой случая отсутствия оценок)
- **Требование**: применение методов `mapToDouble()` и `average()`

#### `getMoviesSortedByAverageRating(boolean descending)`
- Возвращает список фильмов, отсортированных по средней оценке
- Использует Stream API и лямбда-выражения для сортировки
- Параметр `descending` определяет порядок сортировки (по убыванию или возрастанию)
- Фильмы без оценок должны располагаться в конце списка

### 4. Пользовательское исключение `InvalidRatingException`
- Выбрасывается при попытке добавить оценку вне допустимого диапазона
- Должно содержать информацию о некорректном значении

## Технические требования

### Обобщения (Generics)
- Класс `Rating` должен использовать параметр типа `T`, ограниченный `Number`
- Система должна поддерживать различные типы оценок (Integer, Double, Float)
- При расчете среднего значения необходима корректная обработка числовых преобразований

### Stream API
- Расчет средней оценки: `mapToDouble()`, `average()`
- Сортировка фильмов: `stream()`, `sorted()`, кастомный компаратор на основе вычисляемых значений
- Опционально: группировка, фильтрация и другие операции

### Многопоточность
- Гарантия целостности данных при параллельном добавлении оценок
- Оптимальный уровень синхронизации (не блокировать операции чтения без необходимости)
