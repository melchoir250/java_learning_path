# Задача: Потокобезопасная система учета успеваемости студентов

## Описание проекта
Разработайте обобщенную (generic) систему для управления и анализа оценок студентов с поддержкой многопоточного доступа. Система должна обеспечивать типобезопасность, валидацию входных данных и корректную обработку конкурентного доступа к данным.

## Требования к реализации

### 1. Класс `StudentGrade<T extends Number>`
Создайте класс, представляющий запись об оценке студента:
- **Поля**:
   - `String studentName` - имя студента
   - `String subject` - название предмета
   - `T grade` - оценка (параметризованный тип, ограниченный Number)
- **Конструктор**: должен инициализировать все поля класса
- **Методы доступа**: стандартные геттеры для всех полей

### 2. Сервис `GradeService<T extends Number>`
Реализуйте сервис для управления коллекцией оценок:
- **Поле**: `List<StudentGrade<T>> grades` - список для хранения записей
- **Метод `addGrade(StudentGrade<T> grade)`**:
   - Добавляет новую запись об оценке в список
   - Выполняет валидацию: оценка не может быть отрицательной
   - Должен быть потокобезопасным (использовать синхронизацию)
- **Метод `getAverageBySubject(String subject)`**:
   - Рассчитывает среднее арифметическое оценок по указанному предмету
   - Если по предмету нет оценок, метод должен возвращать 0.0
   - Должен корректно работать с любым числовым типом оценок

### 3. Пользовательское исключение `InvalidGradeException`
- Должно выбрасываться при попытке добавить оценку с отрицательным значением
- Должно содержать информативное сообщение об ошибке

### 4. Технические требования

**Обобщения (Generics):**
- Класс `StudentGrade` должен использовать параметр типа `T`, ограниченный `Number`
- Класс `GradeService` должен быть параметризован тем же типом `T`
- Система должна корректно работать с различными типами оценок (`Integer`, `Double`, `Float` и т.д.)

**Потокобезопасность:**
- Метод добавления оценки должен использовать синхронизацию для обеспечения целостности данных при конкурентном доступе
- При расчете среднего значения необходимо учитывать возможные изменения коллекции

**Обработка ошибок:**
- Некорректные (отрицательные) оценки должны отклоняться с выбросом исключения
- Исключение должно содержать детали: какая оценка была отвергнута и почему
