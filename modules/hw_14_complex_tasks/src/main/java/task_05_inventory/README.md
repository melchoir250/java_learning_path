# Задача: Система управления складскими запасами

## Описание проекта

Разработайте систему для управления складом, которая позволяет добавлять и извлекать товары различных категорий. Система
должна контролировать доступ к складским операциям через флаг состояния склада и корректно обрабатывать ситуации
отсутствия товаров.

## Функциональные требования

### 1. Класс `Product`

Создайте класс, представляющий товар на складе:

- **Поля**:
    - `String name` - название товара
    - `double price` - цена товара
    - `String category` - категория товара
- **Конструктор**: для инициализации всех полей
- **Методы доступа**: стандартные геттеры (и опционально сеттеры)
- **Дополнительно**: метод `toString()` для информативного вывода

### 2. Класс `InventoryService`

Реализуйте сервис для управления складскими запасами:

**Состояние:**

- `Map<String, List<Product>> inventory` - структура хранения товаров, где ключ - категория, значение - список товаров в этой категории
- `boolean isInventoryOpen` - флаг, указывающий, открыт ли склад для операций

**Конструктор:**

- Инициализирует пустую карту товаров
- Устанавливает начальное состояние флага (например, `true`)

**Методы:**

#### `void openInventory()`

- Устанавливает флаг `isInventoryOpen` в значение `true`

#### `void closeInventory()`

- Устанавливает флаг `isInventoryOpen` в значение `false`

#### `void addProduct(Product product)`

- Добавляет товар на склад
- **Проверка доступа**: если склад закрыт (`isInventoryOpen == false`), операция не выполняется (можно выводить
  сообщение в консоль или возвращать `false`)
- **Добавление в структуру**: товар добавляется в список соответствующей категории. Если категории еще нет в `Map`, она
  создается

#### `Product takeProduct(String category) throws OutOfStockException`

- Извлекает один товар из указанной категории
- **Проверка наличия**: если категория отсутствует или список товаров в категории пуст, выбрасывается исключение
  `OutOfStockException`
- **Извлечение**: удаляет и возвращает первый товар из списка (или любой другой товар по выбору)

#### `List<Product> findProductsByCategory(String category)`

- Возвращает список всех товаров в указанной категории
- Использует Stream API для фильтрации
- Если категория не найдена, возвращает пустой список

#### `List<Product> filterProductsByPrice(double maxPrice)`

- Возвращает список всех товаров на складе с ценой не выше указанной
- Использует Stream API и лямбда-выражения для фильтрации
- Объединяет товары из всех категорий

#### `List<Product> filterProductsByCategoryAndPrice(String category, double maxPrice)`

- Возвращает список товаров из указанной категории с ценой не выше указанной
- Комбинирует фильтрацию по категории и цене с использованием Stream API

### 3. Пользовательское исключение `OutOfStockException`

- Проверяемое исключение (checked exception)
- Выбрасывается при попытке извлечь товар из пустой категории
- Должно содержать информацию о категории, в которой закончились товары

## Технические требования

### Коллекции

- Использование `HashMap<String, List<Product>>` для хранения товаров по категориям
- Обеспечение корректной работы с вложенными коллекциями (добавление, удаление, поиск)

### Stream API

- Фильтрация товаров по различным критериям (категория, цена)
- Объединение результатов из различных категорий
- Использование лямбда-выражений в операциях `filter()`, `map()`, `collect()`

### Управление состоянием

- Контроль доступа к складским операциям через флаг `isInventoryOpen`
- Изменение состояния склада через специальные методы

### Обработка исключений

- Корректная обработка ситуаций отсутствия товаров
- Информативные сообщения об ошибках
