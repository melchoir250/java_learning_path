# Задача: Многопоточный сервис управления задачами (Task Manager)

## Описание проекта

Разработайте обобщенный сервис для управления задачами, который позволяет добавлять, удалять и выполнять поиск задач по
различным критериям. Система должна обеспечивать уникальность идентификаторов задач, поддерживать потокобезопасные
операции и предоставлять гибкие возможности фильтрации и сортировки с использованием Stream API.

## Функциональные требования

### 1. Класс `Task<T>`

Создайте обобщенный класс, представляющий задачу:

- **Поля**:
    - `T id` - уникальный идентификатор задачи (параметризованный тип)
    - `String description` - описание задачи
    - `TaskStatus status` - статус задачи (перечисление)
    - `Priority priority` - приоритет задачи (перечисление)
    - `LocalDateTime createdAt` - дата и время создания задачи
- **Конструктор**: для инициализации всех полей
- **Методы доступа**: стандартные геттеры для всех полей
- **Методы `equals()` и `hashCode()`**: должны быть реализованы на основе идентификатора

### 2. Перечисления (Enums)

#### `TaskStatus`

- Содержит возможные статусы задачи: `TODO`, `IN_PROGRESS`, `DONE`, `CANCELLED`

#### `Priority`

- Содержит возможные приоритеты задачи: `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`

### 3. Класс `TaskService<T>`

Реализуйте сервис для управления задачами:

**Структура данных:**

- `Map<T, Task<T>> tasks` - хранение задач с возможностью быстрого доступа по идентификатору
- Возможно использование `CopyOnWriteArrayList` или синхронизированной коллекции для потокобезопасности

**Методы:**

#### `void addTask(Task<T> task) throws DuplicateTaskException`

- Добавляет новую задачу в сервис
- **Валидация**: проверяет уникальность идентификатора задачи
- **Исключение**: выбрасывает `DuplicateTaskException`, если задача с таким ID уже существует

#### `void removeTask(T id)`

- Удаляет задачу по идентификатору
- **Потокобезопасность**: метод должен быть синхронизирован для предотвращения конкурентного доступа
- Если задача с указанным ID не существует, метод может выбрасывать исключение или просто завершаться без ошибки (по
  выбору разработчика)

#### `Optional<Task<T>> findTaskById(T id)`

- Ищет задачу по идентификатору
- Возвращает `Optional` для безопасной обработки отсутствующего значения

#### `List<Task<T>> filterTasksByStatus(TaskStatus status)`

- Возвращает список задач с указанным статусом
- Использует Stream API для фильтрации

#### `List<Task<T>> filterTasksByPriority(Priority priority)`

- Возвращает список задач с указанным приоритетом
- Использует Stream API для фильтрации

#### `List<Task<T>> filterTasksByStatusAndPriority(TaskStatus status, Priority priority)`

- Возвращает список задач, соответствующих обоим критериям
- Комбинированная фильтрация с использованием Stream API

#### `List<Task<T>> getTasksSortedByDate(boolean ascending)`

- Возвращает все задачи, отсортированные по дате создания
- Использует `sorted()` с лямбда-выражением для компаратора
- Параметр `ascending` определяет порядок сортировки

#### `List<Task<T>> getTasksSortedByPriority(boolean descending)`

- Возвращает все задачи, отсортированные по приоритету
- Определяет порядок приоритетов (например, CRITICAL > HIGH > MEDIUM > LOW)
- Использует Stream API и кастомный компаратор

### 4. Пользовательские исключения

#### `DuplicateTaskException`

- Проверяемое или непроверяемое исключение (на выбор разработчика)
- Выбрасывается при попытке добавить задачу с уже существующим идентификатором
- Должно содержать информацию о дублирующемся ID

#### `TaskNotFoundException` (опционально)

- Выбрасывается при попытке удалить или найти несуществующую задачу

## Технические требования

### Обобщения (Generics)

- Класс `Task` должен использовать параметр типа `T` для идентификатора
- Класс `TaskService` должен быть параметризован тем же типом
- Система должна поддерживать различные типы идентификаторов (Long, String, UUID и т.д.)

### Коллекции

- Использование `Map` для эффективного доступа по идентификатору
- Обеспечение потокобезопасности при модификации коллекций

### Stream API

- Фильтрация задач с использованием `filter()` по различным критериям
- Сортировка с использованием `sorted()` и кастомных компараторов
- Преобразование результатов с помощью `collect()`

### Лямбда-выражения

- Создание компараторов для сортировки по дате
- Реализация предикатов для фильтрации
- Опционально: использование в других операциях

### Многопоточность

- Синхронизация операций удаления задач
- Обеспечение атомарности операций добавления/удаления
- Корректная обработка итераций по коллекции при параллельных модификациях
