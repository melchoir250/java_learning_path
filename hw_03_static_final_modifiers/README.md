# Домашнее задание: Static, final и модификаторы доступа

## Описание проекта

Проект демонстрирует применение ключевых концепций Java: модификаторов `static`, `final` и различных уровней доступа. Содержит набор классов, каждый из которых focuses на определенных аспектах объектно-ориентированного программирования.

## Классы и их назначение

### Company
- **Статическое поле** `companyName` - общее название компании для всех сотрудников
- **Финальное поле** `employeeID` - неизменяемый уникальный идентификатор сотрудника
- **Методы**: конструктор, геттеры/сеттеры для имени, статический метод вывода названия компании

### MathConstants
- **Финальные поля** `PI` и `E` - математические константы
- **Статические методы** для вычисления площади круга и длины окружности
- Демонстрация использования констант в математических расчетах

### Library
- **Модификаторы доступа**: `private`, `protected`, default, `public`
- **Поля** с разными уровнями доступа: `bookTitle`, `author`, `year`, `category`
- Полный набор геттеров и сеттеров для демонстрации инкапсуляции

### University
- **Статическое поле** `universityName` - общее название университета
- **Финальное поле** `studentID` - неизменяемый идентификатор студента
- **Методы**: статический метод изменения названия, вывод информации о студенте

### GameSettings
- **Статическое поле** `maxPlayers` - общее ограничение игроков
- **Финальное поле** `gameName` - неизменяемое название игры
- **Методы**: управление игроками, проверка ограничений, вывод статуса игры

### Person
- **Приватные поля** с финальным `ssn` - демонстрация строгой инкапсуляции
- **Конструктор** для инициализации всех полей
- **Ограниченные сеттеры** - возможность изменения только имени и фамилии

## Ключевые концепции

### Модификатор Static
- Общие данные для всех экземпляров класса
- Статические методы, не требующие создания объекта
- Изменение статических полей влияет на все объекты класса

### Модификатор Final
- Неизменяемые поля (константы)
- Гарантия постоянства значений после инициализации
- Использование для уникальных идентификаторов

### Модификаторы доступа
- **private** - доступ только внутри класса
- **protected** - доступ в пакете и наследниках
- **default** - доступ только в пределах пакета
- **public** - полный доступ из любого места

## Технические особенности

- Строгое соблюдение принципов инкапсуляции
- Использование финальных полей для иммутабельных данных
- Применение статических методов для утилитарных функций
- Контроль доступа к данным через геттеры и сеттеры

Проект служит практическим примером правильного использования модификаторов в Java для создания надежного и поддерживаемого кода.
